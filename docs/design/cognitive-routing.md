# 思维模式选择

知识大脑不应该只有一种处理问题的方式。

人类面对不同问题时，会自动切换不同思维模式。熟悉的事实可以直接回答，简单查找可以快速搜索，复杂设计需要认真建模，高风险判断需要查证和冲突检测，反复失败的问题则需要反思和学习。

因此，系统需要具备思维模式选择能力。

可以概括为：

```text
不是所有问题都需要完整推理；
系统应该根据问题类型、熟悉程度、不确定性和风险，选择合适的处理模式。
```

## 1. 为什么需要思维模式选择

如果所有问题都走完整知识大脑流程，系统会变得笨重。

例如：

```text
简单事实问题不需要构建完整临时认知模型；
明确文档查找不需要复杂推理；
常见任务不需要每次重新设计路径；
高风险判断不能只靠快速检索；
用户纠正后的问题需要进入反思学习。
```

思维模式选择的目标，是让系统像人一样按需思考。

简单问题快速处理。

复杂问题深度建模。

不确定问题主动查证。

失败问题进入学习。

在工程实现中，思维模式由 agent 承载。

Agent 不是知识本身，也不是单个模型提示词。

Agent 负责按某种思维模式编排多个认知服务。

认知服务负责完成稳定的认知能力，例如问题理解、认知结构激活、补充查找、证据回溯、缺口识别和学习事件记录。

## 2. 模式选择发生在哪里

模式选择发生在问题进入系统之后，深度检索和推理之前。

整体链路变为：

```text
用户问题
  -> 初步理解
  -> 思维模式选择
  -> 对应处理路径
  -> 生成回答
  -> 若产生学习价值则形成 Learning Event 并触发 Study
```

这意味着查找证据、组织临时认知模型、记录 Learning Event 和 Study 不是每次都完整启用。

它们是根据当前模式被选择性调用的能力。

## 3. 如何选择思维模式

系统不应只列出有哪些思维模式，还要说明如何根据问题特征选择模式。

思维模式选择不是让模型自由发挥流程，而是先对问题做认知判定，再映射到合适的处理模式。若模式选择本身暴露学习价值（例如反复误判复杂度），可形成 Learning Event，供 Study 校验选择是否合理。

### 认知判定维度

模式选择应基于一组统一的认知判定维度，而不是单一问题类型标签：

```text
确定性：是否已有稳定答案；
风险：回答错误的代价是否高；
时效性：知识是否可能过期；
证据充分性：已有证据是否足以直接回答；
冲突度：是否存在多来源、多结论冲突；
复杂度：是否需要多变量、多步骤组织；
用户意图：用户是在查找、解释、判断、设计、决策还是反思；
认知预算：本次问题允许投入的认知资源深度。
```

这些维度共同约束模式选择。任何单一维度都不应单独决定模式，但某些组合会强烈指向特定模式。

例如，高确定性、低风险、证据充分，且用户只是在查找或复述，通常不需要工作模型；高时效性、高风险，或冲突度升高，则不应继续停留在直接记忆或快速检索。

### 各模式的触发条件

根据上述维度，各模式的大致触发条件是：

**直接记忆模式**：高确定性、低风险、无需新证据；长期记忆中已有稳定、反复验证的知识；用户意图偏向复述、确认已知结论。

**快速检索模式**：问题明确、证据可直接命中；用户意图偏向查找、定位出处、确认某内容是否存在；复杂度低，冲突度低。

**经验路径模式**：问题属于反复出现的任务类型；已有验证有效的实践路径；场景匹配度高，风险可接受。

**查证模式**：高时效性、高风险，或事实可能变化；证据充分性不足，或用户明确要求依据和来源；不能仅凭长期记忆直接回答。

**工作模型模式**：问题复杂，需要组织变量、证据和推理路径；用户意图偏向解释、判断、设计、决策或多因素综合；证据不足以直接回答，但也不是单纯查找。

**冲突检测模式**：存在互相矛盾的知识、证据或 ActivationLink；多来源给出不同结论；verified 路径与补充查找结果冲突，或 conflicted 链接被命中。

**反思学习模式**：回答失败后触发；用户纠正、连续追问同一缺口、检索失败或推理路径被证明无效；重点不是立刻给出更像答案的回答，而是分析前次处理为何不足。

### 模式升级规则

模式可以在处理过程中升级。常见升级路径是：

```text
快速检索证据不足时，升级为工作模型模式；
发现冲突时，升级为冲突检测模式；
高风险或高时效问题，必须进入查证模式；
回答失败或用户纠正后，进入反思学习模式。
```

此外：

```text
直接记忆发现知识可能过期、用户要求依据或风险升高，应升级为查证或快速检索；
经验路径发现适用条件不满足或出现新情况，应升级为工作模型；
查证发现来源互相矛盾，应升级为冲突检测；
工作模型发现事实基础明显不足，应升级为查证。
```

升级意味着当前模式的输出边界已不够，需要更深层的检索、组织、验证或学习。

### 模式降级规则

模式也可以在条件满足时降级，避免过度推理：

```text
如果工作模型发现问题实际简单、证据已可直接回答，可以降级为快速检索或直接回答；
如果冲突被证据消解、边界已明确，可以回到工作模型或快速检索；
如果查证已确认事实清楚且无需综合判断，可以直接回答或降级为快速检索。
```

降级不是失败，而是说明问题复杂度或不确定性低于最初判断。降级本身不必生成 Trace；仅当降级暴露路径误判或反复模式时，才形成 Learning Event。

### 选择约束

思维模式选择不应完全依赖模型的主观判断。

系统应先用上述认知维度形成约束，再选择模式。例如：

```text
高风险或高时效问题，不得仅因问题表述简单就停留在直接记忆；
存在 conflicted ActivationLink 或明显证据冲突时，不得跳过冲突检测；
candidate 链接不能因模式选择而被当作 verified 路径直接采用；
反思学习应由明确失败信号触发，而不是模型自行切换。
```

模式选择是对问题处理深度的调度，不是对回答风格的偏好。具体模式定义见下一节。

## 认知预算与模式选择

思维模式不仅由问题类型、风险和证据状态决定，还由认知预算决定。

认知预算约束本次问题愿意投入的认知资源：检索深度、KPN 扩展、Working Model 规模、查证与冲突检测强度，以及后续 Study 深度。认知预算用于避免「过度思考」和「无限扩展」。

在模式选择维度中，认知预算与确定性、风险、时效性、证据充分性、冲突度、复杂度和用户意图并列，共同约束模式选择。

### 各模式的默认预算

```text
直接记忆模式：默认 low budget

快速检索模式：默认 low 或 medium budget

工作模型模式：默认 medium 或 high budget

查证模式：默认 medium 或 high budget

冲突检测模式：默认 high budget

反思学习模式：根据影响范围决定 medium 或 high budget
```

默认预算不是固定绑定。系统可在处理过程中根据证据状态和问题演化升级或降级预算，但不应在无必要时超出初始预算所允许的处理深度。

### 预算升级规则

```text
low budget 下如果证据不足，升级到 medium；
medium 下发现高风险、冲突或关键缺口，升级到 high；
high 下仍不能形成可靠结论，应输出边界和缺口，而不是继续无限扩展。
```

预算升级通常伴随模式升级，例如从快速检索进入轻量 Working Model，或从工作模型进入查证或冲突检测。

### 预算降级规则

```text
high budget 问题如果召回证据已经直接充分，可以降级为证据直答；
medium budget 问题如果没有冲突和缺口，可以不构建完整 Working Model。
```

预算降级说明问题复杂度或不确定性低于最初判断。仅在学习价值出现时形成 Learning Event，避免为维持「深度思考」而继续扩散 KPN 或堆叠推理。

### 预算与各环节的约束关系

low budget 下通常不激活 KPN、不构建完整 Working Model，以目录结构召回、全文检索、Rerank 和证据直答为主。

medium budget 下可补充有限 ActivationLink 召回和必要 KPN 上下文，可构建轻量 Working Model。

high budget 下可进入完整 Working Model、冲突检测、外部查证和更完整证据覆盖。

## 4. 主要思维模式

### 直接记忆模式

直接记忆模式适合系统已经高度确定的问题。

例如：

```text
稳定定义；
已经反复确认的个人知识；
无需来源查证的常识性回答；
用户只是要求复述已知结论。
```

这种模式可以直接从长期记忆中回答。

它不需要复杂检索，也不需要构建完整临时认知模型。

但如果问题涉及时效性、风险或不确定性，就不应该使用这种模式。

### 快速检索模式

快速检索模式适合明确查找。

例如：

```text
某个文档中有没有某个观点；
某个概念在哪份材料里出现；
某段原文依据是什么；
用户要求查找而不是分析。
```

这种模式重点是找到相关来源和证据。

它可以轻量组织结果，但不需要完整推理。

### 经验路径模式

经验路径模式适合反复出现、已有成熟处理方法的问题。

例如：

```text
固定操作流程；
常见设计评审；
重复项目复盘；
已经沉淀过的排查步骤。
```

系统应优先调用长期记忆中已经验证有效的实践路径。

这种模式不是从零推理，而是复用过去有效的认知路径。

### 工作模型模式

工作模型模式适合复杂问题。

例如：

```text
系统设计；
方案比较；
投资判断；
复杂解释；
跨文档综合；
多因素决策。
```

这种模式需要激活长期记忆，判断知识缺口，必要时查找外部证据，并组织临时认知模型。

它是知识大脑最完整的思考模式。

### 查证模式

查证模式适合事实可能变化、来源不确定或风险较高的问题。

例如：

```text
今天或最近发生的事情；
政策、价格、版本、人物、公司状态；
用户要求精确来源；
已有知识可能过期；
回答错误代价较高。
```

这种模式的重点不是快速回答，而是确认依据。

系统应该优先查找外部证据，并明确来源和不确定性。

### 冲突检测模式

冲突检测模式适合存在多种观点、证据互相矛盾或结论风险较高的问题。

例如：

```text
支持和反对证据同时存在；
不同文档给出不同判断；
长期记忆和新证据不一致；
用户要求评估利弊。
```

这种模式需要主动寻找反证，并把冲突纳入临时认知模型。

### 反思学习模式

反思学习模式适合系统回答失败或知识缺口暴露之后。

例如：

```text
用户纠正了回答；
用户连续追问同一缺口；
检索没有找到有效依据；
推理路径被证明无效；
某类问题反复处理不好。
```

这种模式的重点不是立刻回答，而是分析失败原因，并生成学习建议。

## 5. 各模式如何工作

不同模式不是同一条流程的不同名称，而是不同的问题处理方式。

每种模式都有自己的目标、过程、输出和记录深度。

### 5.1 直接记忆模式如何工作

直接记忆模式的目标，是快速调用已经稳定的长期记忆。

它适合低风险、低不确定性、系统已经高度熟悉的问题。

处理过程是：

```text
识别问题意图；
确认长期记忆中已有稳定答案；
检查是否存在时效性或风险；
直接组织回答；
通常不产生 Learning Event。
```

这种模式通常不需要外部查证，也不需要构建完整临时认知模型。

它的输出是一个简洁答案。

如果系统发现知识可能过期、用户要求依据、问题风险较高，或者答案不够确定，就应该切换到快速检索或查证模式。

### 5.2 快速检索模式如何工作

快速检索模式的目标，是找到明确材料或证据。

它适合用户要找出处、查原文、确认某个内容是否存在的问题。

处理过程是：

```text
明确查找目标；
确定查找范围；
检索知识单元或来源材料；
返回命中内容和来源；
必要时做轻量总结；
记录查询意图和采用依据。
```

这种模式不应该过度推理。

它的输出通常是来源、摘录、位置、简短说明或直接确认。

如果检索结果之间存在冲突，就切换到冲突检测模式。

如果用户的问题从“查找”变成“解释、比较、判断”，就切换到工作模型模式。

### 5.3 经验路径模式如何工作

经验路径模式的目标，是复用已经被验证有效的实践路径。

它适合反复出现的问题、固定流程、常见任务和已经沉淀的方法。

处理过程是：

```text
识别问题属于已知任务类型；
激活相关实践路径；
检查当前问题是否符合路径适用条件；
按路径组织步骤或建议；
必要时补充少量检索；
记录路径是否有效。
```

这种模式不是从零推理，而是调用长期记忆中已经形成的“怎么做”。

它的输出通常是步骤、流程、检查清单、操作建议或复盘框架。

如果当前问题不符合已有路径的适用条件，或者路径执行中出现新情况，就应该切换到工作模型模式。

如果路径失败，应进入反思学习模式。

### 5.4 工作模型模式如何工作

工作模型模式的目标，是围绕复杂问题构建可推理的临时认知模型。

它适合设计、决策、复杂解释、多因素判断和跨文档综合。

处理过程是：

```text
理解问题；
识别核心变量；
激活相关长期记忆；
判断知识缺口；
必要时查找外部证据；
组织临时认知模型；
检查支持证据和反对证据；
形成推理路径；
输出带边界的回答；
若暴露学习价值则形成 Learning Event。
```

这种模式是最完整的知识大脑流程。

它的输出不是简单答案，而是基于当前认知模型形成的判断、方案、解释或决策建议。

如果发现事实基础不足，应切换到查证模式。

如果发现证据明显冲突，应进入冲突检测模式。

如果回答失败或暴露长期缺口，应进入反思学习模式。

### 5.5 查证模式如何工作

查证模式的目标，是确认事实、来源和时效性。

它适合事实可能变化、来源不确定、风险较高或用户明确要求依据的问题。

处理过程是：

```text
明确需要查证的问题；
列出需要确认的事实点；
查找外部证据或原始来源；
比较来源可靠性和时间有效性；
区分已确认、未确认和存在争议的内容；
给出带来源和不确定性的回答；
记录查证路径。
```

这种模式不应该急于给结论。

它的输出通常包括事实确认、来源说明、时效性说明和不确定性边界。

如果查证结果足够清楚，可以直接回答。

如果查证结果需要综合判断，就切换到工作模型模式。

如果来源之间矛盾，就切换到冲突检测模式。

### 5.6 冲突检测模式如何工作

冲突检测模式的目标，是识别并处理知识、证据或观点之间的矛盾。

它适合多观点、多来源、结论不一致或风险较高的问题。

处理过程是：

```text
识别冲突点；
分别整理支持证据和反对证据；
判断冲突来自事实差异、条件差异、时间差异还是立场差异；
回到来源确认依据；
判断哪些冲突可以解决，哪些必须保留；
把冲突纳入临时认知模型；
输出带条件和边界的回答。
```

这种模式不以消除所有冲突为目标。

有些冲突应该被保留，并在回答中明确表达。

它的输出通常是对比分析、冲突解释、条件化结论或风险提示。

如果冲突解决后需要形成方案，就切换到工作模型模式。

如果冲突暴露长期记忆错误，就进入反思学习模式。

### 5.7 反思学习模式如何工作

反思学习模式的目标，是从失败、纠正和缺口中学习。

它适合用户纠正、回答失败、反复追问、检索失败或推理路径失效之后。

处理过程是：

```text
定位触发反思的事件；
回看问题理解是否正确；
检查检索是否遗漏；
检查临时认知模型是否组织错误；
检查推理路径是否跳跃；
检查长期记忆是否缺失或错误；
形成学习建议；
必要时更新或标记长期记忆。
```

这种模式的重点不是继续生成一个更像答案的答案，而是理解为什么前一次处理不够好。

它的输出通常是学习建议、修正建议、缺口记录、路径调整或后续知识输入需求。

是否立即修改长期记忆，需要谨慎。

如果只是单次反馈，不应轻易重写稳定知识。

如果同类问题反复失败，才应该推动更深层的 知识结构 修正或知识重组。

## 6. 模式选择与推理模板

第 3 节中的认知判定维度决定进入哪种思维模式。问题类型和推理模板则决定在该模式内部如何组织证据和推理。

系统要判断问题是在问：

```text
事实；
定义；
方法；
设计；
判断；
决策；
复盘；
查证；
学习。
```

问题类型影响用户意图和复杂度判断，但不能单独替代模式选择。例如「查证」类问题通常需要查证模式，但一个看似事实型的问题，若高风险或高时效，仍不能停留在直接记忆。

问题类型识别之后、深度检索之前，工作模型等模式还可以生成推理模板计划：

```text
问题 -> 问题类型识别 -> 推理模板选择 -> 生成证据槽位
     -> 按槽位检索和补证 -> 检查模板完整性 -> 执行推理或保守回答
```

一个问题可以组合多个模板，而不是把问题类型与模板一一绑定。例如「为什么超支，还应不应该继续投入」应以 `rule_decision` 为主模板，以 `causal_analysis` 为辅助模板。

常见推理模板包括：

```text
direct_answer：定义、事实、角色和步骤查询；
rule_decision：是否成立、是否应该、能否继续；
causal_analysis：原因、机制和影响；
diagnostic：故障定位和问题排查；
tradeoff：比较、选型和优缺点；
planning：实现规划和目标倒推。
```

冲突不是与上述模板并列的问题类型，而是所有模板都可能进入的控制状态。冲突度升高时，应升级到冲突检测模式；冲突处理完成后可回到原模板；无法消解时输出分支结论和冲突边界。

## 7. ActivationLink 与模式选择

思维模式选择不仅影响检索深度，也影响 ActivationLink 如何被使用。

ActivationLink 不是「被使用过」就会自动稳定。模式选择应结合链接状态和有效性判断，而不是默认沿用所有历史路径。

不同模式对 ActivationLink 的要求不同：

```text
直接记忆 / 快速检索：优先 verified 链接；路径不足时再补充查找；
工作模型：verified 链接提供核心知识，KPN 补充上下文，candidate 不得直接决定推理结构；
查证 / 冲突检测：conflicted 链接必须触发冲突检查；weakened 和 deprecated 链接不应作为当前依据；
反思学习：重点分析 verified 链接是否失效、candidate 是否值得保留、是否存在更优替代路径。
```

模式选择还应参考链接的有效性信号，例如：

```text
答案贡献度：该链接是否真正改善了本次回答；
场景匹配度：当前问题是否落在这条链接的适用边界内；
证据独立性：该链接是否仍有多源、多场景支持；
反例稳定性：相似问题中是否反复出现不适用；
替代路径竞争：是否存在更短、更准的路径；
边界清晰度：系统是否仍明确知道何时不应使用该链接。
```

如果 verified 链接在当前场景中反复不适用，即使历史表现良好，也不应继续优先采用。此时应切换到补充查找、冲突检测或反思学习，而不是强行沿用旧路径。

candidate 链接在任何模式下都不能直接决定答案，只能作为补充探索线索。

verified 链接可以支撑正式召回，但仍需在本次问题处理中持续接受反馈校验，不能视为永久有效。

## 8. 模式不是固定一次

思维模式可以在处理过程中切换。第 3 节已定义升级和降级规则；这里强调切换是正常行为，不是异常回退。

例如：

```text
快速检索发现证据不足，升级为工作模型；
快速检索发现证据冲突，升级为冲突检测；
直接记忆发现知识过期，升级为查证；
工作模型发现问题实际简单，降级为快速检索；
冲突被证据消解后，回到工作模型或快速检索；
回答失败或用户纠正后，进入反思学习。
```

这和人类思考类似。一开始可能只是快速判断，但遇到不确定、冲突或失败信号后，会进入更深层思考或学习。

## 9. 模式与 Learning Event 的关系

不是每种模式都需要生成 Learning Event。

Trace 不是每次问答的必经步骤。只有产生学习价值时才记录事件，例如激活成功或失败、知识缺口、冲突、用户纠正或反复路径信号。

不同模式下，Learning Event 的关注点不同：

```text
直接记忆模式：通常不产生 Learning Event；
快速检索模式：仅在路径成功、失败或用户纠正时记录；
工作模型模式：在缺口确认、路径无效、冲突或用户纠正时记录，不记录完整模型内容；
查证模式：在证据冲突、知识过期或纠正时记录；
冲突检测模式：在稳定冲突或边界变化时记录；
反思学习模式：在用户纠正或反复失败时记录，重点形成学习信号而非复盘叙事。
```

Learning Event 的深度应与学习价值、风险和累积信号匹配，而不是与问题复杂度或推理步骤数量匹配。

## 10. 总结

思维模式选择让知识大脑不是一套固定流程，而是一组可以按需调度的认知能力。

系统应先基于确定性、风险、时效性、证据充分性、冲突度、复杂度和用户意图选择模式，并结合认知预算约束处理深度，再在处理过程中按升级和降级规则切换；选择不应完全依赖模型主观判断。

一句话总结：

```text
知识大脑先判断应该如何思考，再选择是否记忆回答、快速检索、调用经验路径、构建工作模型、查证冲突或进入反思学习；ActivationLink 的使用应服从 verified、candidate、conflicted 等状态边界，而不是被历史命中次数自动放大。
```
