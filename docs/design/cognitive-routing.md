# 思维模式选择

知识大脑不应该只有一种处理问题的方式。

人类面对不同问题时，会自动切换不同思维模式。熟悉的事实可以直接回答，简单查找可以快速搜索，复杂设计需要认真建模，高风险判断需要查证和冲突检测，反复失败的问题则需要反思和学习。

因此，系统需要具备思维模式选择能力。

可以概括为：

```text
不是所有问题都需要完整推理；
系统应该根据问题类型、熟悉程度、不确定性和风险，选择合适的处理模式。
```

## 1. 为什么需要思维模式选择

如果所有问题都走完整知识大脑流程，系统会变得笨重。

例如：

```text
简单事实问题不需要构建完整临时认知模型；
明确文档查找不需要复杂推理；
常见任务不需要每次重新设计路径；
高风险判断不能只靠快速检索；
用户纠正后的问题需要进入反思学习。
```

思维模式选择的目标，是让系统像人一样按需思考。

简单问题快速处理。

复杂问题深度建模。

不确定问题主动查证。

失败问题进入学习。

在工程实现中，思维模式由 agent 承载。

Agent 不是知识本身，也不是单个模型提示词。

Agent 负责按某种思维模式编排多个认知服务。

认知服务负责完成稳定的认知能力，例如问题理解、认知结构激活、补充查找、证据回溯、缺口识别和思考记录。

## 2. 模式选择发生在哪里

模式选择发生在问题进入系统之后，深度检索和推理之前。

整体链路变为：

```text
用户问题
  -> 初步理解
  -> 思维模式选择
  -> 对应处理路径
  -> 必要时记录思考过程
  -> 必要时触发学习
```

这意味着查找证据、组织临时认知模型、记录思考和反馈学习不是每次都完整启用。

它们是根据当前模式被选择性调用的能力。

## 3. 主要思维模式

### 直接记忆模式

直接记忆模式适合系统已经高度确定的问题。

例如：

```text
稳定定义；
已经反复确认的个人知识；
无需来源查证的常识性回答；
用户只是要求复述已知结论。
```

这种模式可以直接从长期记忆中回答。

它不需要复杂检索，也不需要构建完整临时认知模型。

但如果问题涉及时效性、风险或不确定性，就不应该使用这种模式。

### 快速检索模式

快速检索模式适合明确查找。

例如：

```text
某个文档中有没有某个观点；
某个概念在哪份材料里出现；
某段原文依据是什么；
用户要求查找而不是分析。
```

这种模式重点是找到相关来源和证据。

它可以轻量组织结果，但不需要完整推理。

### 经验路径模式

经验路径模式适合反复出现、已有成熟处理方法的问题。

例如：

```text
固定操作流程；
常见设计评审；
重复项目复盘；
已经沉淀过的排查步骤。
```

系统应优先调用长期记忆中已经验证有效的实践路径。

这种模式不是从零推理，而是复用过去有效的认知路径。

### 工作模型模式

工作模型模式适合复杂问题。

例如：

```text
系统设计；
方案比较；
投资判断；
复杂解释；
跨文档综合；
多因素决策。
```

这种模式需要激活长期记忆，判断知识缺口，必要时查找外部证据，并组织临时认知模型。

它是知识大脑最完整的思考模式。

### 查证模式

查证模式适合事实可能变化、来源不确定或风险较高的问题。

例如：

```text
今天或最近发生的事情；
政策、价格、版本、人物、公司状态；
用户要求精确来源；
已有知识可能过期；
回答错误代价较高。
```

这种模式的重点不是快速回答，而是确认依据。

系统应该优先查找外部证据，并明确来源和不确定性。

### 冲突检测模式

冲突检测模式适合存在多种观点、证据互相矛盾或结论风险较高的问题。

例如：

```text
支持和反对证据同时存在；
不同文档给出不同判断；
长期记忆和新证据不一致；
用户要求评估利弊。
```

这种模式需要主动寻找反证，并把冲突纳入临时认知模型。

### 反思学习模式

反思学习模式适合系统回答失败或知识缺口暴露之后。

例如：

```text
用户纠正了回答；
用户连续追问同一缺口；
检索没有找到有效依据；
推理路径被证明无效；
某类问题反复处理不好。
```

这种模式的重点不是立刻回答，而是分析失败原因，并生成学习建议。

## 4. 各模式如何工作

不同模式不是同一条流程的不同名称，而是不同的问题处理方式。

每种模式都有自己的目标、过程、输出和记录深度。

### 4.1 直接记忆模式如何工作

直接记忆模式的目标，是快速调用已经稳定的长期记忆。

它适合低风险、低不确定性、系统已经高度熟悉的问题。

处理过程是：

```text
识别问题意图；
确认长期记忆中已有稳定答案；
检查是否存在时效性或风险；
直接组织回答；
留下轻量思考记录。
```

这种模式通常不需要外部查证，也不需要构建完整临时认知模型。

它的输出是一个简洁答案。

如果系统发现知识可能过期、用户要求依据、问题风险较高，或者答案不够确定，就应该切换到快速检索或查证模式。

### 4.2 快速检索模式如何工作

快速检索模式的目标，是找到明确材料或证据。

它适合用户要找出处、查原文、确认某个内容是否存在的问题。

处理过程是：

```text
明确查找目标；
确定查找范围；
检索知识单元或来源材料；
返回命中内容和来源；
必要时做轻量总结；
记录查询意图和采用依据。
```

这种模式不应该过度推理。

它的输出通常是来源、摘录、位置、简短说明或直接确认。

如果检索结果之间存在冲突，就切换到冲突检测模式。

如果用户的问题从“查找”变成“解释、比较、判断”，就切换到工作模型模式。

### 4.3 经验路径模式如何工作

经验路径模式的目标，是复用已经被验证有效的实践路径。

它适合反复出现的问题、固定流程、常见任务和已经沉淀的方法。

处理过程是：

```text
识别问题属于已知任务类型；
激活相关实践路径；
检查当前问题是否符合路径适用条件；
按路径组织步骤或建议；
必要时补充少量检索；
记录路径是否有效。
```

这种模式不是从零推理，而是调用长期记忆中已经形成的“怎么做”。

它的输出通常是步骤、流程、检查清单、操作建议或复盘框架。

如果当前问题不符合已有路径的适用条件，或者路径执行中出现新情况，就应该切换到工作模型模式。

如果路径失败，应进入反思学习模式。

### 4.4 工作模型模式如何工作

工作模型模式的目标，是围绕复杂问题构建可推理的临时认知模型。

它适合设计、决策、复杂解释、多因素判断和跨文档综合。

处理过程是：

```text
理解问题；
识别核心变量；
激活相关长期记忆；
判断知识缺口；
必要时查找外部证据；
组织临时认知模型；
检查支持证据和反对证据；
形成推理路径；
输出带边界的回答；
记录完整思考过程。
```

这种模式是最完整的知识大脑流程。

它的输出不是简单答案，而是基于当前认知模型形成的判断、方案、解释或决策建议。

如果发现事实基础不足，应切换到查证模式。

如果发现证据明显冲突，应进入冲突检测模式。

如果回答失败或暴露长期缺口，应进入反思学习模式。

### 4.5 查证模式如何工作

查证模式的目标，是确认事实、来源和时效性。

它适合事实可能变化、来源不确定、风险较高或用户明确要求依据的问题。

处理过程是：

```text
明确需要查证的问题；
列出需要确认的事实点；
查找外部证据或原始来源；
比较来源可靠性和时间有效性；
区分已确认、未确认和存在争议的内容；
给出带来源和不确定性的回答；
记录查证路径。
```

这种模式不应该急于给结论。

它的输出通常包括事实确认、来源说明、时效性说明和不确定性边界。

如果查证结果足够清楚，可以直接回答。

如果查证结果需要综合判断，就切换到工作模型模式。

如果来源之间矛盾，就切换到冲突检测模式。

### 4.6 冲突检测模式如何工作

冲突检测模式的目标，是识别并处理知识、证据或观点之间的矛盾。

它适合多观点、多来源、结论不一致或风险较高的问题。

处理过程是：

```text
识别冲突点；
分别整理支持证据和反对证据；
判断冲突来自事实差异、条件差异、时间差异还是立场差异；
回到来源确认依据；
判断哪些冲突可以解决，哪些必须保留；
把冲突纳入临时认知模型；
输出带条件和边界的回答。
```

这种模式不以消除所有冲突为目标。

有些冲突应该被保留，并在回答中明确表达。

它的输出通常是对比分析、冲突解释、条件化结论或风险提示。

如果冲突解决后需要形成方案，就切换到工作模型模式。

如果冲突暴露长期记忆错误，就进入反思学习模式。

### 4.7 反思学习模式如何工作

反思学习模式的目标，是从失败、纠正和缺口中学习。

它适合用户纠正、回答失败、反复追问、检索失败或推理路径失效之后。

处理过程是：

```text
定位触发反思的事件；
回看问题理解是否正确；
检查检索是否遗漏；
检查临时认知模型是否组织错误；
检查推理路径是否跳跃；
检查长期记忆是否缺失或错误；
形成学习建议；
必要时更新或标记长期记忆。
```

这种模式的重点不是继续生成一个更像答案的答案，而是理解为什么前一次处理不够好。

它的输出通常是学习建议、修正建议、缺口记录、路径调整或后续知识输入需求。

是否立即修改长期记忆，需要谨慎。

如果只是单次反馈，不应轻易重写稳定知识。

如果同类问题反复失败，才应该推动更深层的 知识结构 修正或知识重组。

## 5. 模式如何识别

模式识别需要结合几个判断。

### 问题类型

系统要判断问题是在问：

```text
事实；
定义；
方法；
设计；
判断；
决策；
复盘；
查证；
学习。
```

问题类型决定可能的处理模式。

问题类型识别不能直接替代推理方式选择。系统还应在问题理解之后、知识检索之前生成 `ReasoningPatternPlan`：

```text
问题 -> 问题类型识别 -> 推理模板选择 -> 生成证据槽位
     -> 按槽位检索和补证 -> 检查模板完整性 -> 执行推理或保守回答
```

一个问题可以组合多个模板，而不是把问题类型与模板一一绑定。例如“为什么超支，还应不应该继续投入”应以 `rule_decision` 为主模板，以 `causal_analysis` 为辅助模板。

第一版支持六个高频模板：

```text
direct_answer：定义、事实、角色和步骤查询；
rule_decision：是否成立、是否应该、能否继续；
causal_analysis：原因、机制和影响；
diagnostic：故障定位和问题排查；
tradeoff：比较、选型和优缺点；
planning：实现规划和目标倒推。
```

冲突不是与上述模板并列的问题类型，而是所有模板都可能进入的控制状态。冲突处理完成后应回到原模板；无法消解时输出分支结论和冲突边界。

### 知识熟悉度

系统要判断长期记忆中是否已有足够稳定的知识。

如果知识已经反复验证，可以走直接记忆或经验路径。

如果知识很少、很新或很不稳定，就需要检索、查证或工作模型。

### 不确定性

系统要判断当前回答是否存在不确定性。

不确定性可能来自：

```text
知识缺口；
来源不足；
证据冲突；
问题含糊；
时效性强；
用户要求较高精度。
```

不确定性越高，越不应该直接回答。

### 风险和成本

有些问题回答错误代价较高。

例如法律、医疗、金融、生产系统操作、重要决策。

这类问题即使看起来简单，也应该进入查证或工作模型模式。

### 用户意图

用户有时不是要最终答案，而是要：

```text
找资料；
解释依据；
比较方案；
讨论设计；
总结经验；
修正文档；
复盘过程。
```

系统应根据用户意图选择模式。

## 6. 模式不是固定一次

思维模式可以在处理过程中切换。

例如：

```text
快速检索发现证据冲突，切换到冲突检测模式；
直接记忆发现知识过期，切换到查证模式；
工作模型发现反复缺口，切换到反思学习模式；
经验路径无法解决新问题，切换到工作模型模式。
```

这和人类思考类似。

一开始可能只是快速判断，但遇到不确定或冲突后，会进入更深层思考。

## 7. 模式与思考记录的关系

不同模式需要不同程度的思考记录。

直接记忆模式可以只记录轻量思考记录。

快速检索模式应记录查询意图、命中材料和采用依据。

工作模型模式应记录完整的问题理解、知识激活、缺口、证据、模型、推理和反馈。

反思学习模式应重点记录失败原因和学习建议。

思考记录 的深度应该和问题复杂度、风险和学习价值匹配。

## 8. 总结

思维模式选择让知识大脑不是一套固定流程，而是一组可以按需调度的认知能力。

一句话总结：

```text
知识大脑先判断应该如何思考，再选择是否记忆回答、快速检索、调用经验路径、构建工作模型、查证冲突或进入反思学习。
```
