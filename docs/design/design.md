# 知识大脑系统设计

本文档描述如何把 `think.md` 中的总体思想转化为系统设计。

系统的目标不是构建一个静态知识库，而是构建一个能够吸收知识、激活记忆、组织问题、推理、追踪和学习的个人知识大脑。

知识大脑不是通用 Agent 平台，不负责替代 Agent 平台完成任务执行、环境操作和通用工具编排。

知识大脑的目标，是为用户和 Agent 平台提供底层的知识理解、知识使用和知识沉淀能力。

知识大脑也不是知识图谱系统。知识图谱可以作为表达概念、实体和关系的一种方式，但知识大脑的目标不是构建一张完整知识图，而是让知识能够被理解、激活、组织、验证、追踪和沉淀。

整体设计可以概括为：

```text
材料进入
  -> 形成可追溯的知识单元和知识点
  -> 使用中根据问题主题、认知视角和反馈形成候选 ActivationLink
  -> 问题触发思维模式选择（简单问题轻量处理，复杂问题才进入工作模型）
  -> 优先沿 ActivationLink 激活知识点，并反查知识单元和来源
  -> ActivationLink 不足时，按目录结构树、全文检索和外部证据分层补充
  -> 简单问题可直接用召回和 rerank 后的内容组织回答
  -> 复杂问题进入临时认知模型，按思维模板组织推理
  -> 高风险、时效性或证据冲突问题进入查证或冲突检测
  -> 思考记录保存激活、使用、失败和补救过程
  -> 学习机制强化有效 ActivationLink、修正错误路径、发现知识缺口
  -> 长期稳定的知识、经验、主题和解释结构编译为 Wiki 页面
```

知识大脑不是所有问题都走完整推理链路。简单、熟悉、低风险的问题可以轻量处理；复杂设计、判断和决策问题才需要临时认知模型；时效性强、风险高或存在证据冲突的问题，应进入查证或冲突检测。Wiki 页面是长期沉淀结果，不是导入时一次性生成的产物。

## 1. 系统核心对象

### 原始材料

原始材料是外部知识进入系统时的形态。

它可以是文档、笔记、对话、网页、代码、实践记录或其他信息来源。

原始材料不是最终知识，只是知识大脑的输入。

### 知识单元

知识单元是长期记忆的基础编码单位。

它不是为了检索而切出来的文本片段，而是一个相对完整、可理解、可追溯、可复用的知识表达。

一个好的知识单元应该能回答：

```text
它表达了什么知识；
它来自哪里；
它适合在什么问题中被使用；
它能否回到原始材料取证。
```

### 长期记忆

长期记忆是系统已经吸收和组织过的知识。

它由知识单元、概念、知识点、关系、经验、方法、实践路径和证据来源共同构成。

长期记忆不是一次性完成的结构，而是在使用中逐渐稳定。

其中，知识单元和知识点更偏材料侧和证据侧。

领域和概念更偏使用侧和认知侧。

同一批知识单元，在不同使用者、角色、任务类型或长期使用环境下，可以形成不同的领域和概念结构。

### 认知视角

认知视角描述知识大脑当前如何使用知识。

它可以来自：

```text
使用者；
角色；
任务类型；
专业方向；
长期使用环境。
```

例如：

```text
采购管理；
软件架构；
售前投标；
项目管理；
个人学习。
```

领域和概念应在认知视角下理解。

同一个知识单元，在不同认知视角下可以被不同领域和概念激活。

### 知识结构

知识结构描述长期记忆内部如何组织。

它包括：

```text
领域；
概念；
知识点；
关系；
方法；
经验；
实践路径；
待验证假设。
```

知识结构的作用，是让系统面对问题时能够激活相关记忆，而不是从文本中盲目搜索。

知识结构可以包含图状关系，但不等同于知识图谱。

它服务于知识使用，而不是为了构建一张完整静态图。

第一版中的关系主要承担证据、溯源、索引、激活记录和使用记录，不承担开放知识图谱推理。

### 思维模式

思维模式描述系统面对当前问题时应该采用哪种处理方式。

不同问题不应该都走同一条深度推理链路。

系统需要根据问题类型、知识熟悉度、不确定性、风险和用户意图，选择直接记忆、快速检索、经验路径、工作模型、查证、冲突检测或反思学习等模式。

思维模式不是单个模型提示词，也不是某个检索函数。

它应该落地为一类可以被系统选择和执行的问题处理过程。

在工程实现中，思维模式由 agent 承载。

Agent 负责按照某种思维模式编排多个认知服务，完成一次问题处理。

### 认知能力

认知能力描述知识大脑为了理解、检索、组织、推理、追踪和学习而具备的稳定能力。

例如：

```text
问题理解；
认知结构激活；
材料补充查找；
检索结果融合；
缺口识别；
证据回溯；
临时认知建模；
推理组织；
思考记录；
学习沉淀。
```

认知能力不应该直接等同于模型可见的 skill。

它更像系统内部稳定服务。

在工程实现中，认知能力应落地为 service。

Service 负责稳定的数据访问、索引查询、模型调用、错误处理、状态记录和 trace 输出。

模型提示词只参与 service 内部需要语义判断的局部步骤。

因此：

```text
认知能力 -> service；
思维模式 -> agent；
agent 编排 service；
service 在必要时调用模型提示词。
```

### 临时认知模型

临时认知模型是系统围绕本次问题建立的工作模型。

它不是长期记忆，也不是最终回答。

它负责把本次问题相关的概念、知识、证据、缺口、冲突和推理路径组织起来，让系统可以在此基础上思考。

### 思考记录

思考记录描述一次问题处理过程。

它记录问题如何被理解，哪些长期记忆被激活，哪些外部证据被查找，临时认知模型如何形成，推理路径是什么，最终结果和反馈如何。

### 学习结果

学习结果是系统根据思考记录和反馈对长期记忆做出的调整。

它可以强化一个知识点，修正一条关系，补充一个概念，沉淀一条实践路径，或者标记一个知识缺口。

学习结果分属材料层、认知层和表达层。材料层调整知识单元和证据可靠性；认知层调整领域、概念和 ActivationLink；表达层判断哪些主题已足够稳定，适合编译或更新 Wiki 页面。

### Wiki 页面

Wiki 页面是知识大脑在长期使用中逐渐稳定出的主题性知识表达。

它不是原始材料，也不是导入时一次性生成的目录摘要，而是材料层和认知层经过反复激活、验证和反馈后，在表达层编译出的可读、可维护、可演化的长期沉淀。

同一批知识单元在不同认知视角下，可以形成不同的 Wiki 页面。

## 2. 知识如何进入长期记忆

外部材料进入系统后，不应直接成为可使用知识。

它需要先被转换为规范化 Markdown，并保留来源上下文，然后再进入从材料到长期记忆的过程。

### 保留来源

系统首先要保留材料来源。

知识大脑不是凭空记忆，它必须知道每个知识来自哪里。

来源保留的目的不是工程追踪，而是为了让未来回答可以回到证据，并让旧知识在材料变化后可以重新判断。

### 形成知识单元

系统需要把原始材料转换为知识单元。

知识单元的目标不是越短越好，而是表达一个完整的认知动作。

例如：

```text
一个定义；
一个判断；
一个原则；
一个约束；
一种方法；
一个案例；
一个经验。
```

知识单元是长期记忆的最小可用表达。

### 形成初始激活结构

知识单元仍然是局部知识。

系统可以从知识单元中形成轻量的初始激活结构。

它不是完整知识体系，也不是最终 知识结构。

它主要包括：

```text
知识单元中的可激活知识点；
可能匹配的已有领域；
可能匹配的已有概念；
候选领域和候选概念；
来源、证据和使用线索。
```

领域和概念不应该仅因为导入了一篇文档就稳定生成。

它们更多是在真实问题处理中被反复激活、使用、验证和反馈后逐渐稳定下来。

同一批材料，在不同使用者、任务类型和问题场景下，可能形成不同的领域和概念结构。

因此，知识体系不是从导入文件时一次性建立，而是在使用中逐渐长出来。

ActivationLink 也不应在导入阶段稳定生成。它是在真实问题处理中，根据问题主题、认知视角和反馈逐渐形成的认知连接，让相关知识点可以被快速激活，并通过知识点反查知识单元和来源。Wiki 页面同样来自长期使用中的稳定主题，而不是导入时的直接产物。

## 3. 问题如何被处理

当用户提出问题时，系统不应该直接检索文档并生成回答。

它也不应该无论什么问题都进入完整推理链路。

它应该先识别问题需要哪种思维模式，再按对应方式处理。

### 理解问题

系统首先判断问题是什么。

它需要识别：

```text
问题领域；
问题类型；
核心概念；
用户意图；
需要的回答形态；
是否需要外部证据。
```

问题理解决定后续激活哪些长期记忆。

### 选择思维模式

理解问题之后，系统需要选择处理模式。

常见模式包括：

```text
直接记忆；
快速检索；
经验路径；
工作模型；
查证；
冲突检测；
反思学习。
```

模式选择决定后续是否需要激活长期记忆、查找外部证据、组织临时认知模型、记录完整思考过程或触发学习。

知识大脑不是所有问题都完整推理。简单、熟悉、低风险的问题，可以直接基于召回和 rerank 后的内容组织回答；明确查找类问题，可以走快速检索；复杂设计、判断和决策问题，应该进入工作模型模式，按思维模板组织临时认知模型和推理；时效性强、风险高或存在证据冲突的问题，应该进入查证或冲突检测模式；回答失败或用户纠正时，应该进入反思学习模式。

### 分层检索与激活

激活长期记忆不是单一路径，而是分层展开。

系统优先沿使用中形成的 ActivationLink 激活知识点，并通过知识点反查知识单元和来源。当正式激活路径不足时，再依次借助材料目录结构树、全文检索或相近表达检索进行补充；当前知识不足、可能过期或风险较高时，再查找外部证据。

各层召回的结果不应按来源层级简单排序，而应统一比较相关性、证据质量、问题目标、历史使用效果和可追溯性，再决定采用哪些知识进入回答。

### 激活长期记忆

系统根据问题和已选择的模式激活相关知识。

激活不是全文搜索，而是沿着领域、概念、关系、经验和实践路径进行联想。

系统应该优先调用已经形成的长期知识结构。

### 判断知识缺口

系统需要判断已有知识是否足够。

它应该能区分：

```text
可以直接回答的部分；
需要证据确认的部分；
当前知识中缺失的部分；
存在冲突或不确定的部分。
```

这是系统的元认知能力。

### 查找外部证据

当长期记忆不足时，系统应查找外部证据。

外部查找不是随机搜索，而是由当前问题和知识缺口驱动。

它的目标是验证假设、补充事实、确认边界或发现反证。

### 组织临时认知模型

只有复杂问题才需要构建临时认知模型。

当问题足够复杂、存在多因素判断、需要冲突检测，或者简单召回和 rerank 不足以形成可靠回答时，系统需要把激活的长期记忆和查找到的外部证据组织成本次问题的临时认知模型，并按相应思维模板组织推理。

这个模型应该包含：

```text
当前问题；
核心变量；
相关知识；
已知事实；
缺失信息；
支持证据；
反对证据；
可能推理路径；
回答边界。
```

临时认知模型是回答生成前的关键中间层。

### 推理与回答

系统基于临时认知模型完成推理。

推理过程中需要检查：

```text
结论是否有证据支撑；
关系是否成立；
是否存在冲突；
是否需要表达不确定性；
回答是否解决用户问题。
```

最终回答是当前认知条件下的最优输出，而不是绝对真理。

## 4. 思考如何被追踪

系统每次处理问题，都应该留下思考记录，但记录深度应与思维模式匹配。

思考记录不是普通日志。

复杂问题的完整思考记录应该描述：

```text
问题如何被理解；
哪些长期记忆被激活；
发现了哪些知识缺口；
查找了哪些外部证据；
临时认知模型如何组织；
采用了哪些推理路径；
有哪些冲突；
最终回答如何形成；
反馈说明了什么。
```

只有记录思考过程，系统才能知道哪些知识和路径真正有效。

## 5. 知识如何学习和演化

知识大脑必须从使用中学习。

每一次问题处理都会暴露长期记忆的状态：

```text
哪些知识经常被激活；
哪些知识真正参与推理；
哪些关系有效；
哪些路径产生好答案；
哪些知识导致错误；
哪些问题暴露缺口。
```

系统应根据这些信息调整长期记忆。

### 强化

被反复验证有效的知识点、ActivationLink 和实践路径，应该被强化。

### 修正

导致错误理解或错误回答的知识、ActivationLink 或推理路径，应该被修正。

### 补充

反复暴露的缺口，应该成为新知识输入或新学习任务。

### 重组

当旧的知识结构无法解释新的问题模式时，系统应该允许知识结构重组。

### Wiki 编译

当某个主题在材料层和认知层都已足够稳定，表达层应将其编译为 Wiki 页面，或在已有页面上补充、重组和更新。Wiki 编译是学习的结果，不是导入的副产品。

知识演化不是异常流程，而是知识大脑持续成长的正常方式。

## 6. 生命周期与记忆有效性

长期记忆不是永远有效。

知识可能会过期、被修正、被替代或失去上下文。

因此，系统需要管理记忆有效性。

当前回答应该优先使用有效、稳定、可追溯的知识。

旧知识可以保留，用于解释历史思考和知识演化，但不应该默认作为当前答案依据。

生命周期管理在知识大脑中相当于记忆的有效性判断和重新验证机制。

## 7. Agent 如何使用知识大脑

Agent 平台和知识大脑的职责不同。

Agent 平台负责执行任务。

知识大脑负责提供知识、证据、思维模式、临时认知模型、思考追踪和学习沉淀。

因此，Agent 不应该直接访问整个知识仓库，也不应该把知识大脑当成一个普通 RAG 工具。

它应该在明确领域边界内调用知识大脑能力。

知识大脑可以为 Agent 提供：

```text
领域长期记忆；
相关知识激活；
来源证据；
思维模式选择；
临时认知模型；
冲突检测；
思考记录；
经验沉淀和学习建议。
```

Agent 使用知识时，也应该经历类似过程：

```text
理解任务；
选择思维模式；
激活领域记忆；
判断缺口；
必要时组织临时认知模型；
推理执行；
记录思考过程；
反馈学习。
```

Agent 不是知识系统之外的消费者，而是知识大脑的一种实践入口。

Agent 的使用结果也应该反哺长期记忆。

但知识大脑不负责 Agent 平台本身的通用能力，例如：

```text
任务调度；
工具运行时；
浏览器或终端操作；
代码编辑执行；
多 Agent 协作协议；
权限和沙箱管理。
```

这些能力属于 Agent 平台。

知识大脑只关注知识如何被这些任务使用，以及使用之后如何被追踪、评估和沉淀。

## 8. 能力对应关系

系统可以按认知能力组织。

```text
外部材料进入系统；
材料被整理成可追溯的知识单元和知识点；
使用中形成候选 ActivationLink；
问题触发合适的思维模式（简单问题轻量处理，复杂问题进入工作模型）；
优先沿 ActivationLink 激活记忆，不足时分层补充查找；
必要时组织临时认知模型；
思考过程被记录下来；
学习机制强化有效路径、修正错误并发现缺口；
长期稳定的主题编译为 Wiki 页面；
长期记忆的状态和有效性被持续管理。
```

这些能力不是孤立工具，而是知识大脑认知链路中的不同部分。

实现时应避免把认知能力直接做成由模型自由调用的 skill。

认知能力涉及知识库、索引、状态、证据、错误处理和思考记录，应该由稳定 service 承载。

Agent 根据思维模式编排这些 service。

模型只在需要语义判断、摘要、匹配或推理时，作为 service 内部步骤被调用。

## 9. 总结

系统设计的核心不是“文档如何检索”，而是“知识如何被一个持续学习的认知系统使用”。

整体链路是：

```text
材料进入
  -> 形成可追溯的知识单元和知识点
  -> 使用中形成候选 ActivationLink
  -> 问题触发思维模式选择（简单问题轻量处理，复杂问题进入工作模型）
  -> 优先 ActivationLink 激活，不足时分层补充查找
  -> 简单问题直接组织回答，复杂问题进入临时认知模型
  -> 高风险或证据冲突问题进入查证或冲突检测
  -> 记录思考
  -> 学习强化有效路径、修正错误、发现缺口
  -> 长期稳定主题编译为 Wiki 页面
  -> 长期记忆和认知结构持续演化
```

这就是知识大脑系统的基本设计。
