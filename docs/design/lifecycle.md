# 记忆生命周期

长期记忆不是永远有效。

知识会过期、被修正、被替代，也可能失去原来的上下文。

因此，知识大脑需要管理记忆生命周期。

## 1. 为什么需要生命周期

如果系统不区分知识状态，旧知识就可能继续影响当前回答。

这会导致：

```text
过期事实被当成当前事实；
旧判断覆盖新证据；
废弃材料继续支撑回答；
历史推理无法解释当时依据。
```

生命周期管理的目标，是让系统既能使用当前有效知识，也能保留历史解释能力。

## 2. 记忆状态

知识在不同阶段可能处于不同状态。

例如：

```text
当前有效；
候选理解；
需要验证；
已经过期；
被新知识替代；
仅用于历史解释。
```

这些状态会影响知识是否进入当前回答。

## 3. 当前使用和历史解释

当前回答应该优先使用有效、稳定、可追溯的知识。

旧知识不一定删除。

它可以用于解释过去为什么那样回答，也可以用于观察知识如何演化。

但旧知识不应该默认作为当前答案依据。

## 生命周期如何影响激活和 Wiki

生命周期状态不仅影响知识是否进入当前回答，也会影响知识如何被激活、被引用和被沉淀。

具体而言：

```text
知识点是否可被激活；
ActivationLink 是否仍然有效；
目录结构树中的材料是否仍可作为依据；
Wiki 页面是否需要重新编译或标记过期。
```

一条知识单元进入「需要验证」或「已经过期」状态时，其下的知识点不应再被默认激活。依赖这些知识点的 ActivationLink 也应暂停无条件强化，直到重新验证完成。目录结构树保留材料的原有组织，但如果底层来源已失效，目录路径只能帮助定位历史材料，不能自动赋予其当前证据效力。Wiki 页面建立在知识单元、知识点和稳定激活路径之上，底层记忆状态变化时，相关 Wiki 页面也需要重新检查。

当来源材料过期、删除或被替代时，影响会沿证据链向上传导：

```text
相关知识单元需要重新判断；
相关知识点应降低可信度或进入需要验证状态；
依赖这些知识点的 ActivationLink 不应继续无条件强化；
使用这些知识形成的 Wiki 页面需要重新检查。
```

这不是要求立即删除旧内容。旧知识仍可用于解释历史回答、追溯当时依据和观察知识演化过程。但它不能默认作为当前回答依据，也不应继续通过旧 ActivationLink 被优先激活。

生命周期管理的目标不是清理旧内容，而是让系统区分：

```text
当前可用知识；
历史解释知识；
待验证知识；
已被替代知识。
```

当前可用知识可以进入激活、回答和 Wiki 维护。历史解释知识保留解释能力，但不默认参与当前推理。待验证知识应触发重新判断，而不是被继续强化。已被替代知识让位于新证据，旧路径和旧页面应被标记或更新，而不是与新知识并列作为同等依据。

记忆生命周期让知识大脑不仅能学习新知识，也能避免旧知识通过旧激活路径继续误导当前回答。

## 4. 生命周期如何影响学习

当材料更新或知识状态变化时，相关长期记忆也需要重新判断。

例如：

```text
来源过期，相关知识需要重新验证；
新证据出现，旧关系可能需要修正；
实践反馈失败，相关路径可能需要降权；
原始材料删除，相关知识不能继续作为证据。
```

## 5. 总结

生命周期管理是知识大脑的记忆有效性机制。

一句话总结：

```text
记忆生命周期让系统区分当前可用知识和历史知识。
```
