# 初始激活结构

知识单元只是局部记忆。

系统需要让这些局部记忆能够被激活和使用。

但这不意味着导入文件时就应该生成完整知识体系。

更准确地说：

```text
导入阶段形成知识单元和知识点；
使用阶段通过反复激活和反馈逐渐稳定领域和概念；
关系主要承担证据、溯源、索引和使用记录。
```

导入和预编译阶段可以计算 `KnowledgePoint -> Domain / Concept` 的静态匹配候选，但该结果不等于激活路径。`ActivationLink` 描述真实问题场景下的激活条件，只能在使用过程中根据独立召回、实际采用和反馈逐步创建并验证；precompile 不创建 active 或 candidate ActivationLink。

## 1. 为什么不是导入时生成完整知识体系

同一批材料可以服务不同使用者和不同问题场景。

例如同一份招标文件：

```text
采购管理人员会激活招标流程、资格审查、评标办法；
软件架构师会激活办公自动化系统、公文流转、统一用户、信创部署；
售前人员会激活客户需求、评分策略、投标响应、商务条款。
```

材料相同，但使用方式不同，形成的认知结构不同。

因此，知识体系不是材料天然自带的唯一结构，而是材料在使用中被反复激活后形成的认知结构。

## 2. 导入阶段形成什么

导入阶段应该形成相对稳定、可追溯的材料侧知识：

```text
材料来源；
知识单元；
知识点；
来源位置。
```

知识单元是围绕稳定主题或判断形成的最小完整知识包。

知识点是从知识单元中提取的可激活摘要。

来源位置让知识单元和知识点能够回到证据。

这些对象来自材料本身，因此可以在导入阶段形成。

## 3. 使用阶段形成什么

使用阶段形成使用者侧的认知结构：

```text
领域；
概念；
激活权重；
常用检索路径；
实践路径；
候选概念的合并和稳定。
```

领域和概念不应该仅因为某篇文档出现了某个词就变成稳定结构。

它们应该来自：

```text
预制的专业框架；
用户或系统反复使用；
多次思考记录中被激活；
被实际回答采用；
得到反馈强化；
长期稳定存在。
```

## ActivationLink 激活链接

ActivationLink 是知识大脑在使用过程中逐渐形成的认知连接。

它连接的是某一问题主题、认知主题或使用场景，与一组相关知识点。通过知识点可以反查知识单元，再通过知识单元回到规范化 Markdown、预览 HTML 和原始来源。ActivationLink 因此不是孤立的标签挂载，而是能够一路追溯到证据的激活路径。

ActivationLink 不是普通全文索引，也不是一次检索日志。它是长期记忆中可以被学习、强化、降权和修正的激活路径。系统面对问题时，应沿已经验证过的路径激活知识，而不是每次都把 ActivationLink 当成临时搜索结果。

ActivationLink 可以处于候选状态。单次命中、单次采用或单次正反馈，都不足以让它立即稳定。稳定的 ActivationLink 应来自多次思考记录中的反复激活、实际使用、正反馈和持续有效。只有经过反复验证的路径，才值得作为长期认知结构的一部分。

ActivationLink 还应表达使用条件，而不只是表示某个概念下挂载了某个知识点。更准确地说，它表示：在某个认知视角下，当问题处于某类场景、目标、思维模式和注意焦点时，可以通过某个概念激活某个知识点和知识单元。例如，同一个「股票投资」概念下，成长投资场景更关注收入增长和行业空间，价值投资场景更关注估值和现金流，趋势交易场景更关注价格趋势和成交量。如果这些使用条件没有被沉淀，检索阶段只能根据文本相似度临时猜测，相似但不适用的知识容易被激活，历史反馈也难以精确作用到具体路径。

ActivationLink 的价值在于：

```text
让系统面对相似主题时快速激活相关知识；
减少对全文检索的依赖；
帮助系统形成领域和概念下的常用认知路径；
暴露当前认知结构中的缺口；
支持后续 Wiki 页面编译。
```

当某条激活路径反复有效，它会成为领域和概念下的常用认知路径；当问题反复激活失败或只能依赖补充查找才能回答，它也会暴露当前认知结构中的缺口。这些稳定路径和缺口，都是后续 Wiki 编译的重要输入。

ActivationLink 是知识单元、知识点和长期认知结构之间的桥梁。

## 4. 认知视角

同一批知识可以在不同认知视角下形成不同知识体系。

这个视角可以理解为：

```text
使用者；
角色；
任务类型；
专业方向；
长期使用环境。
```

例如：

```text
采购管理；
软件架构；
售前投标；
项目管理；
个人学习。
```

不同认知视角下可以有不同的领域、概念和激活路径。

因此：

```text
知识单元和知识点更偏材料侧和证据侧；
领域和概念更偏使用侧和认知侧。
```

## 5. 认知结构的价值

领域和概念不是为了把知识放进固定分类。

它们的价值，是让系统在面对问题时更会使用知识。

认知结构是知识利用质量的提升层。

它不追求替代补充查找，也不追求一次性整理出完整知识体系。

它通过一次次问题处理，持续提高系统理解问题、筛选材料和组织答案的能力。

认知结构至少提供四种能力：

```text
理解问题；
筛选材料；
组织答案；
发现缺口。
```

当用户提出问题时，系统会先用当前认知视角下已有的领域和概念理解问题。

这种理解会影响系统后续如何查找材料、如何判断结果重要性、如何组织回答。

补充查找可以帮助系统不要漏掉材料。

认知结构则帮助系统判断这些材料应该如何被使用。

因此，领域和概念不是和补充查找竞争的另一套检索结果。

它们是知识使用的框架：

```text
让问题被放到合适的领域中理解；
让相关材料围绕核心概念聚合；
让答案按稳定的判断维度展开；
让补充查找暴露出的缺口可以沉淀为候选概念和候选领域。
```

如果只有补充查找，系统可能找到相关内容，但很难判断哪些是核心、哪些是背景、哪些代表当前认知结构不足。

如果只有认知结构，系统又容易受限于已有领域和概念，漏掉尚未沉淀的新知识。

所以，二者必须互相补充。

从长期看，认知结构的成熟不是表现为材料数量增加，而是表现为知识使用质量提升：

```text
更快理解问题；
更少被相似但无关的材料干扰；
更稳定地找到核心知识；
更清楚地组织答案；
更准确地发现自己缺什么。
```

## 6. 领域

领域是经过预制或反复激活后稳定下来的知识使用范围。

它用于组织记忆、限定问题范围和引导检索。

领域不应该从每篇文档自由生成。

导入阶段最多只能形成候选领域。

稳定领域应来自：

```text
人工预制；
专业框架；
多次使用；
反馈强化；
候选领域合并。
```

## 7. 概念

概念是经过预制或反复激活后稳定下来的知识激活锚点。

它用于连接多个知识单元、知识点和问题场景。

概念不是文档中出现的所有名词。

导入阶段最多只能形成候选概念，或者尝试匹配已有概念。

稳定概念应来自：

```text
专业术语；
核心对象；
系统模块；
关键角色；
流程名称；
指标和公式变量；
反复被问题激活的知识锚点。
```

## 8. 关系

这里的关系不是开放知识图谱关系。

它不负责声明复杂语义关系，例如“支持”“反驳”“导致”“依赖”。

第一阶段的关系主要承担：

```text
证据；
溯源；
索引；
激活记录；
使用记录。
```

也就是说，它记录：

```text
知识点来自哪个知识单元；
知识单元来自哪个材料；
某个问题激活了哪些知识；
某次回答实际使用了哪些知识；
某个候选概念是从哪些使用记录中观察到的。
```

这些关系记录知识从哪里来、什么时候被激活、是否被使用，而不是试图构建完整知识图谱。

## 9. 总结

初始激活结构的目标，不是导入时整理完全部知识体系。

它的目标是让知识单元和知识点可以被使用，并为后续使用中的学习留下线索。

一句话总结：

```text
知识单元来自材料，知识体系来自使用；同一材料可以在不同认知视角下长出不同领域和概念。
```
