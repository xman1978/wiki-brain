# 反馈学习

知识大脑必须从使用中学习。

学习不是简单添加新文档，而是根据真实问题处理结果调整长期记忆。

学习也不是把导入材料预编译成固定知识体系。

它是一个闭环：

```text
当前领域和概念尝试理解问题；
系统从材料和证据中补充查找；
思考记录保存激活、使用、失败和补救过程；
学习从记录中识别知识缺口和认知缺口；
候选领域、候选概念和思考路径被强化、降权、合并或晋升。
```

## 1. 学习来自哪里

学习主要来自思考记录。

每一次思考记录都会暴露：

```text
哪些知识被激活；
哪些知识真正参与推理；
哪些证据有效；
哪些领域和概念激活有效；
哪些补充查找弥补了激活失败；
哪些候选领域和候选概念被观察到；
哪些路径产生好回答；
哪些地方导致错误；
哪些知识缺口反复出现。
```

这些信息比静态文档更接近真实知识价值。

## 2. 学习什么

系统不只学习事实。

它还应该学习：

```text
概念是否重要；
证据、溯源和激活关系是否可靠；
路径是否有效；
问题类型如何识别；
哪些证据值得信任；
哪些知识需要重新验证；
哪些缺口值得补充。
```

其中，领域和概念尤其应该从使用中学习。

同一批材料在不同使用场景下，会被不同问题反复激活，从而形成不同的领域边界和概念锚点。

因此：

```text
领域和概念不应该仅由导入文件时自动生成；
它们可以被预制，也可以从思考记录中的反复激活和实际使用中逐渐稳定；
被召回但未使用的候选概念不应自动强化；
被多次使用并得到正反馈的概念和路径，才适合强化。
```

系统还应该学习激活路径的适用条件。

也就是说，学习不只是判断：

```text
这条 ActivationLink 是否有效。
```

还要判断：

```text
这条 ActivationLink 在什么场景下有效；
服务什么目标时有效；
配合哪种思维模式时有效；
关注哪些问题焦点时有效；
指向的知识点是否真正支撑了本次问题；
类似问题中的历史反馈是否稳定。
```

因此，思考记录中的学习信号应该尽量保留这些认知维度：

```text
场景；
目标；
思维模式；
注意焦点；
知识点相关性；
正向反馈；
负向反馈；
未使用原因；
边界不匹配原因。
```

## 学习结果的层次

学习不是一个单一动作。

同一次问题处理可能同时暴露材料、认知和表达三个层面的信号，学习结果也应分层理解：

```text
材料层学习；
认知层学习；
表达层学习。
```

三层学习都依赖思考记录，但调整的对象不同，演化速度也不同。

### 材料层学习

材料层学习调整的是知识单元、知识点和来源证据本身是否可靠。

它包括：

```text
知识单元是否可靠；
知识点是否表达准确；
来源和证据是否有效；
某些知识是否需要重新验证、降权或替换。
```

当材料更新、证据失效、回答被纠正或某条知识反复导致误解时，学习应落在材料层。这对应修正、降权和补充：修正不准确的表达，降权不可靠的知识，补充反复暴露的事实缺口。

### 认知层学习

认知层学习调整的是系统如何理解问题、激活记忆和组织检索路径。

它包括：

```text
领域和概念是否被正确激活；
ActivationLink 是否有效；
常用检索路径是否值得强化；
实践路径是否可复用；
某些候选概念是否应该合并、拆分、晋升或降权。
```

认知层学习主要作用于领域、概念、ActivationLink 和实践路径。强化、降权和重组大多发生在这里：反复有效的激活路径被强化，边界不清的概念被拆分或合并，候选结构在多次验证后晋升或废弃。

### 表达层学习

表达层学习调整的是哪些知识已经适合沉淀为面向人阅读和维护的长期表达。

它包括：

```text
哪些主题已经适合沉淀为 Wiki 页面；
哪些 Wiki 页面需要补充、重组或更新；
哪些经验可以从多次问题处理中提炼出来；
哪些解释结构更适合人阅读和复用。
```

表达层学习不直接修改底层证据，而是判断某个主题是否已在材料层和认知层足够稳定，值得编译为 Wiki 页面，或已有 Wiki 页面是否需要更新。Wiki 编译是表达层学习的结果，而不是导入或单次问答的副产品。

### 学习信号的使用原则

三层学习共享同一套谨慎原则，与强化、修正、降权、补充、重组保持一致：

```text
被召回但未使用的知识不应自动强化；
被实际使用并得到正反馈的路径才适合强化；
单次失败不应直接重写长期记忆；
多次稳定信号才适合推动认知结构和 Wiki 页面演化。
```

一次检索命中只说明候选知识值得观察，不说明它应该被强化。一次回答失败可能来自问题理解、证据不足或推理错误，不应立即推翻整条 ActivationLink 或整份 Wiki 页面。只有当同类信号在多次思考记录中反复出现，学习才应推动材料修正、认知结构调整或 Wiki 重新编译。

## KPN 与 ActivationLink 学习

KPN 本身不直接形成稳定激活路径。

KPN 中的知识点连接，只说明知识点之间存在轻量语义关系，不说明它们适合在某类问题中被优先激活。

ActivationLink 必须来自真实问题处理中的采用和反馈。

因此：

```text
KPN 连接 ≠ ActivationLink；
KPN 扩散命中 ≠ 有效激活路径；
KPN 上下文被采用 ≠ 立即晋升为 ActivationLink。
```

当某些 KPN 上下文在多次问题中被证明有效时，Study 可以生成 candidate ActivationLink。

但 candidate ActivationLink 仍必须经过独立补充查找、实际采用、证据回链和反馈验证，才能晋升为 verified。

## ActivationLink 质量控制

ActivationLink 不是「被使用过」就会自动稳定。

一次命中、一次采用或一次正反馈，只说明这条路径值得观察，不说明它已经具备长期激活资格。Study 对 ActivationLink 的目标，不是生成更多链接，而是筛选、验证、降权和淘汰链接。

### 有效性判断标准

一条 ActivationLink 是否值得保留或晋升，应综合以下标准判断：

```text
答案贡献度：没有该链接，答案质量是否明显下降；
场景匹配度：该链接是否只在特定问题类型、任务场景、领域上下文中有效；
证据独立性：是否由不同来源、不同知识单元、不同使用场景共同支持；
反例稳定性：是否在相似问题中多次被证明不适用；
替代路径竞争：是否存在更短、更准确、更稳定的激活路径；
边界清晰度：是否明确知道该链接适用和不适用的条件。
```

只有多条标准在多次思考记录中共同支持，链接才适合从 candidate 向 verified 晋升。单次成功不足以跳过验证。

### 状态模型

ActivationLink 应处于明确的生命周期状态中：

```text
candidate：候选链接，只能辅助探索；
verified：已验证链接，可参与正式召回；
weakened：被降权链接；
conflicted：存在冲突的链接；
deprecated：不再推荐使用的链接。
```

各状态的含义是：

candidate 链接不能直接决定答案，只能作为补充检索线索。它不得进入正式召回排序，不得单独支撑最终回答。

verified 链接可参与正式召回，但仍需持续接受反馈校验，不能视为永久有效。当反例累积、边界失效或出现更优替代路径时，verified 链接应被降权、标记冲突或废弃。

weakened 链接仍可被记录和解释，但不应再作为优先激活路径。

conflicted 链接表示同一问题上下文下，与其他 verified 路径、证据或边界判断存在未解冲突，需要进入冲突检测后再决定是否采用。

deprecated 链接保留历史解释能力，但不应再参与当前回答和 Wiki 维护。

### candidate 与 verified 的边界

```text
candidate 不能直接决定答案，只能辅助探索；
verified 可以参与正式召回，但不能永久免审；
weakened、conflicted、deprecated 都不应作为当前首选激活路径。
```

Study 的主要职责，是根据思考记录中的采用、失败、反例和替代路径信号，推动链接在以上状态间迁移，而不是让链接数量持续增长。

## 3. 如何学习

学习可以表现为几种变化。

### 强化

反复有效的知识、证据关系、激活关系和路径应该变得更容易被激活。

对领域和概念来说，强化不来自单次出现，而来自多次被激活、被使用并得到正反馈。

对激活链接来说，强化也不应该只是增加全局权重。

更准确地说，系统应该强化某条链接在特定认知维度下的适用性：

```text
某类场景；
某类目标；
某种思维模式；
某些注意焦点；
某类问题表达；
某组知识点相关性。
```

例如，一条链接在“性能问题 + 故障排查模式 + 根因定位目标”下多次有效，不代表它在“系统设计 + 方案规划目标”下也应该被强化。

### 修正

导致错误的问题理解、证据关系、激活关系或推理路径应该被修正。

如果一条激活链接被反复召回但不适合当前问题，系统需要判断错误来自哪里：

```text
场景识别错误；
目标识别错误；
思维模式选择错误；
注意焦点不匹配；
知识点相关性不足；
概念边界过宽；
链接适用边界过宽。
```

修正不一定意味着删除链接。

更常见的是收窄它的适用条件，或者把它拆成更精确的激活路径。

### 降权

经常被召回但很少被使用的知识，或者经常导致误解的知识，应该被降低重要性。

经常匹配问题但无法激活有效知识的领域和概念，也应该被降权或重新审视边界。

激活链接的降权也应尽量作用在具体维度上。

例如：

```text
在某个场景下降权；
在某个目标下降权；
在某种思维模式下降权；
在某类注意焦点下降权。
```

只有当一条链接在多个维度下持续无效、缺少证据支撑或边界明显错误时，才应整体降为 weakened 或 deprecated。

### 补充

反复出现的知识缺口应该形成新的学习目标。

### 重组

当旧知识结构无法支持真实问题时，长期记忆结构应该允许重组。

重组可以包括：

```text
候选概念晋升为正式概念；
候选领域晋升为正式领域；
已有概念拆分；
相近概念合并；
概念移动到更合适的领域；
某个认知视角下形成新的激活路径；
按场景、目标、思维模式和注意焦点拆分或合并激活路径。
```

当补充查找找到高质量知识，并且该知识在多次问题处理中被采用时，学习可以构建候选激活链接。

候选激活链接是待验证的学习假设，不是低权重检索路径。它不得参与正式激活、召回排序或答案知识选择。验证它的材料必须来自独立的补充查找、实际采用结果、证据回链和反馈，不能由候选链接自身制造命中信号。

但候选激活链接不能只包含领域、概念和知识点。

它还应该带上从真实使用中观察到的认知维度：

```text
这个候选链接是在什么场景中出现的；
它服务了什么目标；
当时使用了什么思维模式；
问题关注点是什么；
该知识点为什么相关；
有哪些正反馈和负反馈；
它的适用边界是什么。
```

这样后续晋升的不是一条粗糙的「概念到知识点」连接，而是一条有适用条件的激活规则。

候选链接经过有效性标准检验后，才可能晋升为 verified；持续无效、边界错误或存在更优替代路径时，应转为 weakened 或 deprecated。存在未解冲突时，应标记为 conflicted，待冲突检测和学习后再决定去向。

candidate 链接不能直接决定答案，只能作为补充检索线索。它不通过累计候选召回次数逐渐获得正式召回权，也不以 candidate 状态长期替代 verified 路径。

## 4. 学习需要谨慎

不是每次反馈都应该立即改变长期记忆。

一次失败可能来自问题表达、证据不足、推理错误，也可能来自知识本身错误。

系统应该先理解失败原因，再决定如何学习。

## 5. 总结

反馈学习让知识大脑不是越存越大，而是越用越准。

对 ActivationLink 而言，Study 的目标不是链接越多越好，而是让 verified 链接更少、更准、边界更清晰。

一句话总结：

```text
学习从思考记录中提取信号，并用它修正长期记忆。
```
