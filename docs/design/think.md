# 知识大脑总体思想

这个系统不是一个死的知识存储系统。

它的目标也不是把文档保存起来，然后在用户提问时检索几个相似片段。

更准确地说，它应该是一个个人知识大脑：能够吸收外部知识，形成可追溯的知识单元和知识点；在使用中根据问题主题、认知视角和反馈形成 ActivationLink；优先沿激活路径召回相关知识，并在不足时通过目录结构、全文检索和外部证据分层补充；面对简单问题轻量处理，面对复杂问题组织临时认知模型；生成回答后，仅在产生学习价值时记录 Learning Event，并由 Study 调整长期记忆；最终将长期稳定的知识、经验和主题编译为 Wiki 页面。

知识大脑的核心不是把文档预编译成固定知识体系，也不是把所有问题都走完整推理链路。

它的核心是一个闭环：

```text
材料进入
  -> 形成可追溯的知识单元和知识点
  -> 使用中形成或验证 ActivationLink
  -> 问题触发思维模式选择（简单问题轻量处理，复杂问题才进入工作模型）
  -> ActivationLink 激活知识，不足时按目录结构树、全文检索和外部证据分层补充
  -> KPN 在必要时补充上下文
  -> Working Model 在复杂问题中组织思考
  -> 生成回答

如果本次问答产生学习价值：
  -> 形成 Learning Event
  -> Study 根据 Learning Event 调整长期记忆
  -> 长期稳定的知识、经验、主题和解释结构编译为 Wiki 页面
```

因此，知识单元来自材料，ActivationLink 和 Wiki 页面来自使用。

当前认知结构和补充查找不是竞争关系。

补充查找让系统不要漏掉材料。

认知结构让系统更会理解问题、筛选材料、组织答案，并判断自己的认知结构哪里需要强化。

认知结构检索的核心价值，不是替代补充查找，而是持续提升知识利用质量。

知识大脑的成长，不只是材料越来越多，而是同样的材料被越来越好地使用。

同一批材料，在不同使用者、任务类型和问题场景下，可以形成不同的领域和概念。

一句话概括：

```text
wiki 不是知识仓库，而是一个能围绕问题思考、验证、学习，并把稳定知识逐步编译为可读 Wiki 页面的知识大脑。
```

## 1. 为什么不是普通知识库

普通知识库通常围绕保存和查找展开：

```text
保存文档
切分片段
检索内容
生成回答
```

这种方式的问题是，它把知识看成静态材料，把检索看成搜索，把回答看成片段拼接。

但人类使用知识时并不是这样。

人不会在大脑里全文搜索所有记忆，然后直接输出答案。人也不会遇到任何问题都完整推理一遍。人会先判断这个问题需要怎样处理：熟悉事实可以直接回答，明确查找可以快速检索，常见任务可以调用经验路径，复杂问题才需要建立临时认知模型，高风险问题则需要查证和冲突检测。

因此，本系统不应该只模拟搜索系统，而应该模拟人的知识使用过程。

## 2. 为什么不是 Agent 平台

知识大脑也不是通用 Agent 平台。

Agent 平台更关注任务执行，例如任务分解、工具调用、环境操作、代码修改、浏览网页、多 Agent 协作和运行时管理。

知识大脑关注的是另一件事：

```text
知识如何进入长期记忆；
知识如何被问题激活；
知识体系如何在使用中形成；
认知缺口如何从补充查找中暴露；
知识如何被组织成本次问题的认知模型；
知识如何参与推理；
Knowledge Brain 如何在产生学习价值时记录 Learning Event；
知识如何在 Study 中沉淀和演化。
```

也就是说，知识大脑不是替代 Agent 平台，而是为 Agent 平台提供底层的知识能力和思维能力。

Agent 平台负责做事。

知识大脑负责知道、理解、思考、追踪和学习。

二者的关系可以概括为：

```text
Agent 平台
  -> 调用知识大脑获得领域长期记忆、证据、思维模式和临时认知模型；

知识大脑
  -> 从 Agent 的使用过程和反馈中沉淀经验，持续更新长期记忆。
```

这个边界很重要。

知识大脑不应该发展成通用任务执行平台，也不应该负责所有工具调度和环境操作。

它的核心目标，是让知识被更好地使用、追踪和沉淀。

## 3. 为什么不是知识图谱

知识大脑也不是知识图谱系统。

知识图谱强调实体、关系、属性、三元组和图查询。

这些能力可以帮助表达知识结构，但它们不能定义知识大脑本身。

知识大脑的目标不是构建一张完整、正确、稳定的知识图。

它的目标是让知识能够在问题处理中被激活、组织、验证、追踪和沉淀。

也就是说：

```text
知识图谱关注知识如何被结构化表示；
知识大脑关注知识如何被理解、使用和学习。
```

知识大脑可以使用知识图谱作为一种底层表示或索引方式。

但它不应该被简化成知识图谱。

## 4. 人如何使用知识解决问题

人在处理问题时，不是只有一条固定链路，而是会先选择合适的思维模式。

简单问题可能直接调用记忆。

明确查找可能只需要快速检索。

复杂问题才会进入完整的理解、查证、建模、推理和反馈过程。

因此，下面描述的是复杂问题中的完整认知链路，而不是所有问题都必须执行的固定流程。

### 选择思维模式

人在面对问题时，会先粗略判断它需要怎样处理。

例如：

```text
这是我已经确定知道的事实吗？
这是只需要查找出处的问题吗？
这是一个熟悉的任务流程吗？
这是需要多因素判断的复杂问题吗？
这个问题是否有时效性或高风险？
是否存在互相矛盾的说法或证据？
这次失败是否说明我需要学习？
```

这个判断决定后续是直接回答、快速查找、调用经验路径，还是进入深度推理、查证、冲突检测或反思学习。

在系统中，这种判断会落地为思维模式选择。

思维模式不是模型自己临时发挥出的流程，也不是完全依赖模型主观偏好，而是由一组统一的认知判定维度约束：

```text
确定性：是否已有稳定答案；
风险：回答错误的代价是否高；
时效性：知识是否可能过期；
证据充分性：已有证据是否足以直接回答；
冲突度：是否存在多来源、多结论冲突；
复杂度：是否需要多变量、多步骤组织；
用户意图：用户是在查找、解释、判断、设计、决策还是反思。
```

根据这些维度，系统选择直接记忆、快速检索、经验路径、查证、工作模型、冲突检测或反思学习。处理过程中还可以升级或降级：证据不足时从快速检索进入工作模型，发现冲突时进入冲突检测，高风险或高时效问题必须查证，回答失败或用户纠正后进入反思学习；若工作模型发现问题实际简单，则可降级为快速检索或直接回答。

思维模式由 agent 承载，负责按当前模式编排内部认知服务。

认知服务承载具体认知能力，例如问题理解、认知结构激活、补充查找、证据回溯、缺口识别和学习事件记录。

### 理解问题

首先不是找答案，而是判断问题是什么。

例如：

```text
这是事实问题，还是判断问题？
这是技术问题，还是决策问题？
它属于哪个知识领域？
它真正想解决什么？
```

这一步会激活长期记忆中的知识结构。

### 激活已有知识

人会从长期记忆中联想相关知识。

这种激活不是全文搜索，而是通过概念、经验、关系和场景扩散。

在使用中，系统还会沿 ActivationLink 这类已经验证过的认知路径，快速激活相关知识点，并通过知识点反查知识单元和来源。当这些路径不足时，再借助材料目录结构、全文检索或外部证据分层补充。

例如，“创业板现在能买吗”可能激活：

```text
投资决策
成长股
估值
利率
盈利增速
风险偏好
历史市场周期
```

### 判断自己是否知道得足够

人还会判断：

```text
哪些我确定知道；
哪些我大概知道但不确定；
哪些我明显不知道；
哪些我可能不知道自己不知道。
```

这是一种元认知能力。

没有元认知，系统就容易把不完整知识当成确定答案。

### 查找外部证据

当已有知识不足时，人会借助外部信息。

但这种查找不是随机搜索，而是带着假设搜索。

已有知识会帮助人提出更具体的问题：

```text
当前估值处于什么位置？
盈利是否恢复？
利率环境是否变化？
有没有相反证据？
```

外部搜索不是替代内部知识，而是用来验证、补充和修正当前理解。

### 组织临时认知模型

人不会把所有记忆和资料都堆在一起。

面对一个具体问题，人会临时组织一个当前问题专用的认知模型。

这个模型会保留：

```text
问题是什么；
关键变量有哪些；
已有知识能解释什么；
还缺什么证据；
可能的推理路径是什么；
支持和反对证据分别是什么；
最终应该如何表达答案。
```

这个临时认知模型不是长期记忆，也不是最终回答。它是本次思考的工作台。

### 推理和冲突检测

有了临时认知模型后，人会进行推理。

推理可能是演绎、归纳、类比，也可能是反事实分析。

同时，人会寻找冲突：

```text
有没有反对证据？
有没有条件不成立？
有没有遗漏关键变量？
有没有不同解释？
```

冲突会推动人重新查找证据、修正模型，直到形成当前最合理的判断。

### 输出当前最优答案

人的回答通常不是绝对真理，而是当前认知条件下的最优答案。

有些答案是确定的，有些是概率性的，有些是带风险边界的。

好的知识系统也应该能区分这些情况，而不是把所有回答都伪装成同等确定。

### 反馈学习

问题结束后，如果有反馈，人会更新长期记忆。

正确的推理路径会被强化。

错误的判断会被修正。

暴露出的知识缺口会变成后续学习目标。

这就是知识从实践中生长。

## 5. 知识大脑的核心能力

基于上面的认知过程，系统需要的不只是存储和检索，而是一组连续能力。

### 长期记忆

长期记忆保存系统已经吸收和组织过的知识。

它不是原始文档集合，而是由知识单元、概念、关系、经验、方法、实践路径和来源证据共同构成的知识体系。

### 内部激活

面对问题时，系统应该先激活已有知识。

它需要知道问题可能属于哪个领域，相关概念是什么，已有经验能提供哪些线索。

优先路径是 ActivationLink：它携带使用中形成的主题关联，让相关知识点被快速激活，并沿知识单元回到来源。ActivationLink 不足时，系统再借助目录结构树、全文检索和外部证据分层补充。

这对应人的联想式记忆激活，而不是每次从全文重新搜索。

### 元认知

系统需要判断自己知道什么，也要知道自己不知道什么。

如果已有知识足够，就可以直接基于长期记忆推理。

如果已有知识不足，就应该明确缺口，并查找外部证据。

### 思维模式选择

系统需要根据问题特征选择合适的处理方式，而不是让所有问题都进入完整推理链路。

模式选择应基于统一的认知判定维度：

```text
确定性、风险、时效性、证据充分性、冲突度、复杂度、用户意图。
```

据此映射到不同模式：

```text
直接记忆：高确定性、低风险、无需新证据；
快速检索：问题明确、证据可直接命中；
查证：高时效性、高风险或事实可能变化；
工作模型：问题复杂，需要组织变量、证据和推理路径；
冲突检测：存在互相矛盾的知识、证据或 ActivationLink；
反思学习：回答后发现路径、知识、链接需要调整。
```

模式可以在处理过程中升级或降级。快速检索证据不足时可进入工作模型；发现冲突时进入冲突检测；高风险或高时效问题必须查证；回答失败或用户纠正后进入反思学习；若工作模型发现问题实际简单，可降级为快速检索或直接回答；冲突被证据消解后可回到工作模型或快速检索。

模式选择不应完全依赖模型主观判断，而应由上述认知维度形成约束。

### 外部查证

外部查证用于补充和验证长期记忆。

它不是盲目搜索，而是由当前问题和临时认知模型提出证据需求。

### 临时认知模型

临时认知模型组织本次问题所需的知识、证据、变量、冲突和推理路径。

它是长期记忆和最终回答之间的中间层。

没有临时认知模型，系统容易退化成把检索结果直接拼成回答。

### 推理与冲突检测

系统需要基于临时认知模型完成推理，并主动检查支持证据和反对证据。

冲突不是异常，而是高质量思考的一部分。

### 学习事件记录

Knowledge Brain 不追求保存完整思考过程。

系统只在出现学习价值时记录 Learning Event。Learning Event 关注：

```text
激活路径是否成功；
是否存在认知缺口；
是否出现用户纠正；
是否发现知识冲突；
是否有候选 ActivationLink；
是否需要调整 Concept 或 Wiki；
是否需要重新验证某个 KnowledgePoint。
```

人类长期记忆通常不会保存每次思考的完整过程，而更倾向保存结果、经验、错误、冲突和可复用路径。

Knowledge Brain 模拟的是知识经验积累，不是完整人类反思能力。

完整复盘和反思属于 Agent Runtime，不属于 Knowledge Brain 的核心职责。

### 反馈学习

系统需要从 Learning Event 中学习。

学习不是简单增加内容，而是强化有效 ActivationLink，修正错误路径，补充缺口，重组长期记忆，并判断哪些主题已适合编译或更新 Wiki 页面。

### Wiki 编译

Wiki 页面不是导入时一次性生成的目录摘要，而是表达层长期沉淀的结果。

当某个主题在多次问题处理中被反复激活、实际采用并得到正反馈，其相关知识、经验和解释结构才适合被编译为 Wiki 页面。Wiki 让知识大脑不只能够回答问题，还能形成面向人阅读和维护的长期知识体系。

## 6. 总体原则

### 知识不是静态材料

知识进入系统后，不应该只停留为文档。

它要被编码成长期记忆，并在使用中不断被验证和修正。

### 检索不是搜索片段

检索的本质，是问题对长期记忆的激活，以及对外部证据的定向查找。

### 回答不是拼接资料

回答应该来自本次问题的临时认知模型。

系统需要先组织问题、变量、证据、关系和推理路径，再输出答案。

### Learning Event 不是完整过程记录

Learning Event 不是检索日志，也不是 Chain-of-Thought，更不是每次问答的必经步骤。

它只记录对长期记忆学习有价值的事件和结果。只有记录这些学习信号，Study 才知道后续应该强化、降权或重组什么。

### 学习不是批处理

学习发生在知识使用之后。

真实问题会暴露哪些 ActivationLink 有效、哪些路径错误、哪些材料需要补充、哪些主题适合沉淀为 Wiki 页面。

系统应该从这些实践中更新长期记忆，而不是在导入时一次性整理出完整知识体系。

### 简单问题与复杂问题分流

知识大脑不是一个固定工作流。

它应该根据认知判定维度选择不同思维模式，而不是默认进入完整推理：

```text
直接记忆、快速检索、经验路径、查证、工作模型、冲突检测、反思学习，都是知识大脑的正常能力。
```

简单、熟悉、低风险且证据充分的问题可以轻量处理；复杂、多变量、需要综合判断的问题才需要临时认知模型；高风险、高时效或冲突度高的问题应进入查证或冲突检测；回答失败或用户纠正后应进入反思学习。

### 知识大脑不是 Agent 平台

知识大脑可以服务 Agent，但不承担通用 Agent 平台职责。

它不以任务执行、工具编排和环境操作为核心。

它以长期记忆、知识使用、学习事件记录和知识沉淀为核心。

### 知识大脑不是知识图谱

知识图谱可以作为知识结构的一种表达方式。

但知识大脑的重点不是静态表示实体和关系，而是让知识在问题处理中被激活、组织、验证、追踪和沉淀。

## 7. 总结

知识大脑的核心链路是：

```text
外部知识输入
  -> 形成可追溯的知识单元和知识点
  -> 使用中形成或验证 ActivationLink
  -> 面对问题时选择思维模式（简单问题轻量处理，复杂问题进入工作模型）
  -> ActivationLink 激活知识，不足时分层补充查找
  -> KPN 在必要时补充上下文
  -> Working Model 在复杂问题中组织思考
  -> 生成回答

如果本次问答产生学习价值：
  -> 形成 Learning Event
  -> Study 根据 Learning Event 调整长期记忆
  -> 长期稳定主题编译为 Wiki 页面
```

这个系统的目标不是让知识被存起来，而是让知识被使用、被检验、被修正，并逐渐形成更好的个人知识大脑。
