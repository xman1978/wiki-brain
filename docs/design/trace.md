# 思考追踪

思考记录不是检索日志。

它记录的是知识大脑处理一次问题时的思考过程。

## 1. 为什么需要思考记录

如果系统只保存最终回答，就无法知道：

```text
问题是否被正确理解；
哪些长期记忆被激活；
哪些领域和概念参与了激活；
当前认知结构在哪里失效；
哪些证据被采用；
哪些补充查找结果补救了激活失败；
临时认知模型如何形成；
推理路径是否合理；
回答失败时问题出在哪里。
```

没有思考记录，系统就无法真正学习。

## 2. 思考记录记录什么

一次完整的思考记录应该记录：

```text
用户问题；
问题理解；
当前认知视角；
匹配到的领域和概念；
激活的长期记忆；
发现的知识缺口；
发现的认知缺口；
查找的外部证据；
补充查找结果；
哪些补充查找结果弥补了领域和概念激活失败；
临时认知模型；
推理路径；
冲突检测；
最终回答；
用户或系统反馈；
候选领域、候选概念和新的思考路径；
后续学习建议。
```

## 3. 召回和使用要分开

思考记录必须区分：

```text
哪些知识被召回；
哪些知识真正被使用。
```

被召回说明它进入候选范围。

被使用说明它参与了临时认知模型、推理路径或最终回答。

这个区别决定后续学习方向。

思考记录还必须区分：

```text
哪些结果来自当前领域和概念的激活；
哪些结果来自补充查找；
两类结果如何被共同比较和采用；
哪些补充查找结果最终被采用；
哪些采用结果暴露了领域和概念缺口。
```

被召回但未使用的知识，不应该自动强化。

领域和概念激活失败但补充查找成功，是重要的学习信号。

领域和概念激活结果较弱，而补充查找结果更好，也应该被记录。

这说明系统不一定缺少材料，但可能缺少更准确的概念、领域边界或激活路径。

## 4. 思考记录如何服务学习

思考记录让系统可以判断：

```text
哪些知识应该强化；
哪些激活路径应该强化；
哪些关系只是证据和使用记录；
哪些路径值得沉淀；
哪些概念边界不清；
哪些领域和概念缺失；
哪些知识缺口需要补充。
```

思考记录是学习的输入。

候选领域和候选概念不应该在一次思考记录后立即稳定。

它们需要在多次思考记录中被反复激活、使用和反馈后，才适合晋升为正式认知结构。

## 5. 总结

思考记录的核心价值，是把一次回答变成一次可学习的思考样本。

一句话总结：

```text
思考记录记录知识大脑如何理解、激活、组织、推理和修正。
```
