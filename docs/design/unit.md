# 知识单元

知识单元是知识大脑形成长期记忆的基础编码单位。

它的核心定义是：

```text
知识单元是围绕一个稳定主题或判断形成的最小完整知识包。
```

知识单元不是知识点。

知识点可以是一句话、一个断言、一个局部观察；知识单元则需要围绕一个稳定中心组织必要的解释、依据、边界和上下文。

知识单元划分和知识点提取属于同一次材料理解。对于上下文窗口足够的模型，导入阶段应通过一次主模型调用联合产出 `KnowledgeUnit[]` 及各单元内的 `KnowledgePoint[]`，避免先划分单元、再逐单元重复理解。结构校验失败时只允许对失败单元做局部修复，不应重跑整份材料。

它不是为了检索而切出来的文本块，而是一个可以被理解、引用、组合、追溯和学习的知识表达。

知识单元不作为单独文件存储。

它来自规范化 Markdown，并通过数据库记录它在 Markdown 中的位置。

## 1. 为什么需要知识单元

原始材料通常是连续文本。

连续文本适合阅读，但不适合直接成为长期记忆。

知识大脑需要把材料编码成更小、更清晰、更可复用的单位。

知识单元的作用是：

```text
保留原文依据；
表达一个完整知识动作；
支持后续概念和关系形成；
支持问题处理时被激活；
支持在思考记录中被引用；
支持反馈后被强化或修正。
```

## 2. 什么是好的知识单元

好的知识单元不是越短越好。

它应该表达一个完整的认知动作。

例如：

```text
一个概念定义；
一个判断；
一个原则；
一个约束；
一种方法；
一个案例；
一个经验；
一个问题。
```

如果切得太碎，知识会失去语义完整性。

如果切得太粗，知识又会接近普通文档片段，难以被组合和学习。

## 3. 如何形成知识单元

知识单元应从原文结构和语义共同形成。

推荐三段式流程：

```text
结构 / 目录划定范围（source、recovered、semantic outline segment）
  → 材料知识维度分析（该范围内有几个可独立表达的知识面）
  → 边界裁决（accept / split / merge / discard）
  → 一个稳定 center 对应一个 KnowledgeUnit
```

可以概括为：

```text
目录划定范围；
维度决定有几个独立知识面；
边界裁决确定最小完整知识包。
```

标题、段落、列表、引用、表格和代码块提供可追溯的原文结构。

source outline、recovered outline 和 semantic outline 提供目录上下文和初始范围，但不直接等同于知识单元。

材料知识维度分析判断：在某个 outline segment 内，材料实际包含几个可独立表达、可单独引用的知识面，例如定义、规则、约束、流程、参数表、案例等。

边界裁决再判断每个知识面是否已构成最小完整知识包，是否需要拆分、合并或丢弃。

最终判断标准是：这个单元是否围绕一个稳定 center（主题或判断）形成了最小完整知识包，将来能否作为长期记忆的一部分被使用。

### 维度不等于目录节点，也不等于检索维度

以下映射都不应默认成立：

```text
outline 节点 ≠ 知识单元；
材料维度 ≠ 知识单元；
outline 节点 ≠ 材料维度。
```

材料知识维度只描述原文材料内部有几个独立知识面，服务于 unit 切块。

它与 retrieval 中的认知路由维度（场景、目标、模式、焦点）无关，也与 precompile 中 KnowledgePoint 的激活标签无关。

同一个 outline segment 可能包含多个材料维度；多个 outline 节点也可能共同支撑一个材料维度；某些目录节点只是过渡结构，不应生成知识单元。

## 4. 知识单元与来源

每个知识单元都必须能回到原始材料。

知识大脑不能只保存抽象结论，也要保存结论来自哪里。

在存储上，知识单元应记录自己在规范化 Markdown 中的位置，例如所属文档、标题路径、起止位置和必要的来源上下文。

系统通过这些位置信息回到 Markdown，再通过来源信息回到预览 HTML 和原始材料。

来源追溯的价值在于：

```text
回答时可以取证；
知识过期时可以重新判断；
思考记录可以解释当时依据；
反馈学习可以定位具体知识。
```

## 5. 知识单元与长期记忆

知识单元本身还不是完整长期记忆。

它提供材料侧、证据侧的稳定知识基础。

知识单元可以生成可激活的知识点，并在后续使用中被反复召回、采用、验证和反馈。

领域和概念不应该仅从单篇材料中直接稳定生成。

它们会在使用过程中，围绕不同使用者、任务类型和问题场景逐渐形成。

同一批知识单元，在不同使用方式下可能形成不同的领域和概念结构。

但没有高质量知识单元，长期记忆就没有可靠基础。

## 6. 总结

知识单元是原始材料变成长期记忆的第一步。

一句话总结：

```text
知识单元不是单独保存的文本切片，而是记录在 Markdown 位置上的可追溯、可复用、可学习的最小完整知识包。
```
