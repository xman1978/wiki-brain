# 临时认知模型

临时认知模型是知识大脑处理单个问题时临时建立的工作结构。

它不是长期记忆，也不是最终答案。

它的作用，是把本次问题相关的长期知识、外部证据、缺口、冲突和推理路径组织起来，让系统能够围绕当前问题思考。

临时认知模型不是所有问题都需要。

只有当问题足够复杂、存在不确定性、需要多因素判断、需要冲突检测，或者回答风险较高时，系统才应该构建它。

可以概括为：

```text
长期记忆回答“我知道什么”；
检索回答“这个问题激活了什么”；
临时认知模型回答“本次问题应该如何被组织和思考”；
思考记录 回答“这次思考实际如何发生”。
```

## 1. 为什么需要临时认知模型

如果没有临时认知模型，系统很容易退化为：

```text
检索一些材料；
把材料放进上下文；
生成一个看似合理的回答。
```

这不是思考。

真正的问题处理，需要先组织出本次问题的认知结构：

```text
问题是什么；
关键变量是什么；
已有知识能解释什么；
缺少什么；
证据支持什么；
证据反对什么；
应该如何推理；
回答边界在哪里。
```

临时认知模型就是这个结构。

但对简单事实、明确查找和稳定记忆问题，构建完整临时认知模型反而会增加不必要成本。

因此，临时认知模型应该由思维模式选择触发，而不是所有问题的默认路径。

## 2. 与工作模型相关的模式选择

临时认知模型对应工作模型模式，但不是所有问题都应进入该模式。

系统应先基于认知判定维度选择模式：

```text
确定性：是否已有稳定答案；
风险：回答错误的代价是否高；
时效性：知识是否可能过期；
证据充分性：已有证据是否足以直接回答；
冲突度：是否存在多来源、多结论冲突；
复杂度：是否需要多变量、多步骤组织；
用户意图：用户是在查找、解释、判断、设计、决策还是反思。
```

**进入工作模型模式**的典型条件是：复杂度较高，需要组织变量、证据和推理路径；用户意图偏向解释、判断、设计或决策；证据不足以直接回答，但也不是单纯查找。

**不应进入工作模型模式**的典型情况是：高确定性、低风险、证据已可直接回答，且用户只是在查找或复述。此时应使用直接记忆或快速检索。

**模式升级**：快速检索发现证据不足、问题从查找转为解释或判断、或需要多因素综合时，应升级为工作模型。高风险或高时效问题若在工作模型中暴露事实基础不足，应进一步升级为查证；发现冲突时应升级为冲突检测。

**模式降级**：如果工作模型组织过程中发现问题实际简单、召回证据已可直接回答，可以降级为快速检索或直接回答，而不必维持完整推理结构。若冲突被证据消解，也可从冲突检测回到工作模型或快速检索。

模式选择不应完全依赖模型主观判断。是否构建临时认知模型，应服从上述认知维度和升级、降级规则，而不是因为模型倾向于「多想一步」就默认进入工作模型。

## 3. 它来自哪里

临时认知模型来自三类信息。

### 用户问题

用户问题提供目标和约束。

系统需要理解用户是在问事实、方法、判断、解释、决策，还是要求设计。

思维模式选择会判断这个问题是否需要工作模型。

### 长期记忆

长期记忆提供已有知识。

系统会激活相关领域、概念、知识点、关系、经验和实践路径。

### KnowledgePoint Network 提供上下文

Working Model 不直接从 KPN 中搜索答案。

KPN 的作用，是在核心 KnowledgePoint 已经被召回之后，补充本次问题可能需要的上下文。

这些上下文可以包括：

```text
相关变量；
前置概念；
边界条件；
反例；
补充知识；
对立知识；
可能缺失的知识点。
```

Working Model 使用 KPN 的结果来完善问题框架、核心变量、证据集合和知识缺口。

因此：

```text
KPN 负责补充上下文；
Working Model 负责组织思考。
```

### 外部证据

当长期记忆不足时，系统需要查找外部证据。

这些证据用于验证假设、补充事实、发现反例或确认边界。

## 4. 它包含什么

一个临时认知模型至少应包含几类内容。

### 问题框架

问题框架说明系统如何理解当前问题。

例如：

```text
这是一个设计问题；
这是一个投资判断问题；
这是一个事实查证问题；
这是一个方法选择问题。
```

### 核心变量

核心变量是影响回答的关键因素。

例如，投资问题可能包含估值、盈利、利率、政策、风险偏好。

系统需要知道哪些变量重要，而不是把所有相关信息平铺出来。

### 已知知识

已知知识来自长期记忆。

它描述系统当前已经可以使用的概念、原则、经验和关系。

### 知识缺口

知识缺口描述系统当前不知道或不确定的部分。

缺口会决定是否需要查找外部证据。

### 证据集合

证据集合包括支持证据和反对证据。

系统应该避免只收集支持自己初始判断的信息。

### 推理路径

推理路径描述系统准备如何从知识和证据走向回答。

它可以是因果路径、比较路径、步骤路径、类比路径或反事实路径。

### 回答边界

回答边界说明本次回答的确定性和适用范围。

有些回答可以确定，有些回答只能给出概率，有些回答必须附带条件和风险。

## 5. 它如何工作

临时认知模型不是一次性生成后就不变。

它会在问题处理过程中被不断修正。

```text
理解问题
  -> 选择工作模型模式
  -> 初始模型
  -> 激活长期记忆
  -> 发现缺口
  -> 查找证据
  -> 修正模型
  -> 推理
  -> 发现冲突
  -> 再修正模型
  -> 形成回答
```

如果新证据推翻了原有判断，临时认知模型应该被更新。

如果发现关键变量缺失，系统应该补充变量。

如果冲突无法解决，回答中应该保留不确定性。

## 临时认知模型质量检查

临时认知模型组织完成后、输出回答前，系统应对其做质量检查。一个好的 Working Model 必须满足以下标准：

**1. 问题边界清晰**

明确本次问题要回答什么、不回答什么。不能把相邻问题、背景信息或用户未询问的判断混入当前模型。

**2. 关键变量完整**

覆盖影响判断的主要变量。遗漏关键变量会导致推理看似完整，实则建立在片面结构上。

**3. 证据覆盖充分**

每个关键判断都应有对应证据。不能把推断、类比或未验证假设当作已确认事实。

**4. 知识缺口显式**

不能把未知内容伪装成已知。缺口应被明确标记，并决定是否需要补充查找、查证或保守回答。

**5. 支持与反对证据并存**

不能只收集支持当前结论的材料。Working Model 应主动保留反例、边界条件和反对证据。

**6. 推理路径可检查**

结论应能回溯到变量、证据和知识单元。若无法说明某个结论来自哪些证据和推理步骤，该结论不应进入最终回答。

**7. 回答边界匹配证据强度**

证据弱时只能给出保守结论。不能把低置信度推断表达为确定判断，也不能忽略时效性、冲突和适用边界。

**8. 不直接沉淀为长期知识**

Working Model 是一次性思考结构，随本次问题结束而消散。它不能直接写入长期记忆、不能直接晋升为 ActivationLink 或 Wiki 页面；只有经过 Trace 记录和 Study 验证后，才可能影响长期记忆。

质量检查未通过的模型，应回到补充检索、冲突检测或模型修正，而不是强行生成看似完整的回答。

## 6. 与其他模块的关系

### 与知识检索的关系

知识检索为临时认知模型提供被激活的长期记忆和外部证据。

但检索结果不是临时认知模型本身。

检索结果必须经过问题级 rerank、结构化证据聚合、覆盖检查和冲突检查，才能进入临时认知模型。

临时认知模型接收的不是原始召回片段，而是已经绑定证据的事实、规则、实体关系、变量和缺口。它在此基础上构建候选推理路径，并明确区分：

```text
文档明确规定的事实和规则；
可以从事实和规则推导出的判断；
缺少证据或量化标准的部分；
因冲突或覆盖不足而无法确定的结论。
```

### 与思考记录的关系

思考记录记录临时认知模型如何形成、如何变化、如何支撑回答。

如果回答失败，思考记录可以帮助判断是问题理解错了、检索不足、证据错误，还是推理路径有问题。

### 与学习的关系

学习从临时认知模型和思考记录中发生。

如果某种模型结构反复有效，它可以沉淀为实践路径。

如果某种模型经常失败，说明长期记忆或问题建模方式需要修正。

## 7. 总结

临时认知模型是知识大脑的工作记忆。

它让系统不是简单调用知识，而是围绕当前问题组织知识。

一句话总结：

```text
临时认知模型把长期知识和外部证据组织成本次问题可推理、可检查、可追踪的工作结构。
```
