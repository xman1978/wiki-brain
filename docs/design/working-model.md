# 临时认知模型

临时认知模型是知识大脑处理单个问题时临时建立的工作结构。

它不是长期记忆，也不是最终答案。

它的作用，是把本次问题相关的长期知识、外部证据、缺口、冲突和推理路径组织起来，让系统能够围绕当前问题思考。

临时认知模型不是所有问题都需要。

只有当问题足够复杂、存在不确定性、需要多因素判断、需要冲突检测，或者回答风险较高时，系统才应该构建它。

可以概括为：

```text
长期记忆回答“我知道什么”；
检索回答“这个问题激活了什么”；
临时认知模型回答“本次问题应该如何被组织和思考”；
Learning Event 记录“本次处理对长期记忆有何学习价值”。
```

## 1. 为什么需要临时认知模型

如果没有临时认知模型，系统很容易退化为：

```text
检索一些材料；
把材料放进上下文；
生成一个看似合理的回答。
```

这不是思考。

真正的问题处理，需要先组织出本次问题的认知结构：

```text
问题是什么；
关键变量是什么；
已有知识能解释什么；
缺少什么；
证据支持什么；
证据反对什么；
应该如何推理；
回答边界在哪里。
```

临时认知模型就是这个结构。

但对简单事实、明确查找和稳定记忆问题，构建完整临时认知模型反而会增加不必要成本。

因此，临时认知模型应该由思维模式选择触发，而不是所有问题的默认路径。

## 2. 与工作模型相关的模式选择

临时认知模型对应工作模型模式，但不是所有问题都应进入该模式。

系统应先基于认知判定维度选择模式：

```text
确定性：是否已有稳定答案；
风险：回答错误的代价是否高；
时效性：知识是否可能过期；
证据充分性：已有证据是否足以直接回答；
冲突度：是否存在多来源、多结论冲突；
复杂度：是否需要多变量、多步骤组织；
用户意图：用户是在查找、解释、判断、设计、决策还是反思。
```

**进入工作模型模式**的典型条件是：复杂度较高，需要组织变量、证据和推理路径；用户意图偏向解释、判断、设计或决策；证据不足以直接回答，但也不是单纯查找。

**不应进入工作模型模式**的典型情况是：高确定性、低风险、证据已可直接回答，且用户只是在查找或复述。此时应使用直接记忆或快速检索。

**模式升级**：快速检索发现证据不足、问题从查找转为解释或判断、或需要多因素综合时，应升级为工作模型。高风险或高时效问题若在工作模型中暴露事实基础不足，应进一步升级为查证；发现冲突时应升级为冲突检测。

**模式降级**：如果工作模型组织过程中发现问题实际简单、召回证据已可直接回答，可以降级为快速检索或直接回答，而不必维持完整推理结构。若冲突被证据消解，也可从冲突检测回到工作模型或快速检索。

模式选择不应完全依赖模型主观判断。是否构建临时认知模型，应服从上述认知维度和升级、降级规则，而不是因为模型倾向于「多想一步」就默认进入工作模型。

### 认知预算约束

Working Model 的构建规模受认知预算约束：

```text
low：不构建 Working Model，或只做极轻量判断；
medium：构建轻量 Working Model，只保留必要变量和证据；
high：构建完整 Working Model，允许冲突检测、反例查找和外部查证。
```

low budget 下若证据已充分，应优先证据直答，不构建完整 Working Model。high budget 下仍无法形成可靠结论时，应输出保守结论和缺口，而不是继续扩展 KPN 或堆叠推理。

### Working Model 扩展停止规则

当以下任一条件满足时，应停止向 Working Model 补充知识和上下文：

```text
新增知识点不能改变问题理解、判断边界或结论强度；
证据不足时，应标记缺口，而不是继续扩展；
认知预算耗尽时，应输出保守结论和缺口。
```

停止扩展不等于回答失败。它说明当前证据和预算下，继续扩散不会提高回答可靠性，应明确回答边界和待补缺口。

## 3. 它来自哪里

临时认知模型来自三类信息。

### 用户问题

用户问题提供目标和约束。

系统需要理解用户是在问事实、方法、判断、解释、决策，还是要求设计。

思维模式选择会判断这个问题是否需要工作模型。

### 长期记忆

长期记忆提供已有知识。

系统会激活相关领域、概念、知识点、关系、经验和实践路径。

### KnowledgePoint Network 提供上下文

Working Model 不直接从 KPN 中搜索答案。

KPN 的作用，是在核心 KnowledgePoint 已经被召回之后，补充本次问题可能需要的上下文。

这些上下文可以包括：

```text
相关变量；
前置概念；
边界条件；
反例；
补充知识；
对立知识；
可能缺失的知识点。
```

Working Model 使用 KPN 的结果来完善问题框架、核心变量、证据集合和知识缺口。

因此：

```text
KPN 负责补充上下文；
Working Model 负责组织思考。
```

Working Model 可以请求 KPN 补充上下文，但 KPN 补充必须围绕当前 Working Model 的变量、边界、反例和缺口。Working Model 不能把 KPN 当成无限知识扩展工具。

KPN 的激活条件、扩展边界和停止条件详见 `docs/design/kpn.md`。当认知预算不足或 KPN 停止条件满足时，应停止扩展并保留已获得的上下文。

### 外部证据

当长期记忆不足时，系统需要查找外部证据。

这些证据用于验证假设、补充事实、发现反例或确认边界。

## 4. 它包含什么

一个临时认知模型至少应包含几类内容。

### 问题框架

问题框架说明系统如何理解当前问题。

例如：

```text
这是一个设计问题；
这是一个投资判断问题；
这是一个事实查证问题；
这是一个方法选择问题。
```

### 核心变量

核心变量是影响回答的关键因素。

例如，投资问题可能包含估值、盈利、利率、政策、风险偏好。

系统需要知道哪些变量重要，而不是把所有相关信息平铺出来。

### 已知知识

已知知识来自长期记忆。

它描述系统当前已经可以使用的概念、原则、经验和关系。

### 知识缺口

知识缺口描述系统当前不知道或不确定的部分。

缺口会决定是否需要查找外部证据。

### 证据集合

证据集合包括支持证据和反对证据。

系统应该避免只收集支持自己初始判断的信息。

### 推理路径

推理路径描述系统准备如何从知识和证据走向回答。

它可以是因果路径、比较路径、步骤路径、类比路径或反事实路径。

### 回答边界

回答边界说明本次回答的确定性和适用范围。

有些回答可以确定，有些回答只能给出概率，有些回答必须附带条件和风险。

## 5. 它如何工作

临时认知模型不是一次性生成后就不变。

它会在问题处理过程中被不断修正。

```text
理解问题
  -> 选择工作模型模式
  -> 初始模型
  -> 激活长期记忆
  -> 发现缺口
  -> 查找证据
  -> 修正模型
  -> 推理
  -> 发现冲突
  -> 再修正模型
  -> 形成回答
```

如果新证据推翻了原有判断，临时认知模型应该被更新。

如果发现关键变量缺失，系统应该补充变量。

如果冲突无法解决，回答中应该保留不确定性。

## 临时认知模型质量检查

临时认知模型组织完成后、输出回答前，系统应对其做质量检查。一个好的 Working Model 必须满足以下标准：

**1. 问题边界清晰**

明确本次问题要回答什么、不回答什么。不能把相邻问题、背景信息或用户未询问的判断混入当前模型。

**2. 关键变量完整**

覆盖影响判断的主要变量。遗漏关键变量会导致推理看似完整，实则建立在片面结构上。

**3. 证据覆盖充分**

每个关键判断都应有对应证据。不能把推断、类比或未验证假设当作已确认事实。

**4. 知识缺口显式**

不能把未知内容伪装成已知。缺口应被明确标记，并决定是否需要补充查找、查证或保守回答。

**5. 支持与反对证据并存**

不能只收集支持当前结论的材料。Working Model 应主动保留反例、边界条件和反对证据。

**6. 推理路径可检查**

结论应能回溯到变量、证据和知识单元。若无法说明某个结论来自哪些证据和推理步骤，该结论不应进入最终回答。

**7. 回答边界匹配证据强度**

证据弱时只能给出保守结论。不能把低置信度推断表达为确定判断，也不能忽略时效性、冲突和适用边界。

**8. 不直接沉淀为长期知识**

Working Model 是一次性思考结构，随本次问题结束而消散。它不能直接写入长期记忆、不能直接晋升为 ActivationLink 或 Wiki 页面。只有 Working Model 暴露的学习价值经 Learning Event 记录、Study 处理后，才可能影响长期记忆。

质量检查未通过的模型，应回到补充检索、冲突检测或模型修正，而不是强行生成看似完整的回答。

## 6. 与其他模块的关系

### 与知识检索的关系

知识检索为临时认知模型提供被激活的长期记忆和外部证据。

但检索结果不是临时认知模型本身。

检索结果必须先经问题级 Rerank 逐条分类，再经结构化证据聚合、覆盖检查和冲突检查，才能进入临时认知模型。

Rerank 是逐条证据分类器，不是 Top-K 选择器：它只判断每条候选证据是否与问题匹配，以及它是直接证据、支持性证据还是无关证据，并将所有直接证据和支持性证据交给后续环节；它不理解完整回答结构，也不决定是否需要原因或推理步骤。临时认知模型接收的不是原始召回片段，也不是 Rerank 的分类结果本身，而是聚合与检查之后已经绑定证据的事实、规则、实体关系、变量和缺口。它在此基础上构建候选推理路径，并明确区分：

```text
文档明确规定的事实和规则；
可以从事实和规则推导出的判断；
缺少证据或量化标准的部分；
因冲突或覆盖不足而无法确定的结论。
```

### 与 Learning Event 的关系

Working Model 不默认生成完整过程 Trace。

Working Model 内部如何组织变量、修正推理、逐步推导，不属于 Learning Event 的记录范围。

若 Working Model 暴露了激活失败、知识缺口、路径冲突或用户纠正等学习信号，可形成 Learning Event。Learning Event 记录的是学习事实，不是 Working Model 的完整内容。

### 与学习的关系

Study 的输入是 Learning Event，不是 Working Model 本身。

如果某类知识组织路径在多次 Learning Event 中反复被证明有效，Study 可沉淀为实践路径。如果某类 Working Model 反复暴露同类失败信号，Study 应调整 ActivationLink、Concept 或知识状态，而不是保存 Working Model 的完整结构。

更完整的链路如下：

```text
复杂问题进入工作模型模式
  -> Working Model 组织本次变量、证据和推理路径
  -> 若暴露学习价值，形成 Learning Event（如 activation_success、repeated_success）
  -> Study 从多次事件中观察结构相似性（任务类型、核心变量组合、步骤框架）
  -> 提炼候选实践路径
  -> 经独立验证后，成为经验路径模式可调用的长期记忆模板
```

Working Model 只贡献学习信号，不直接晋升为实践路径。单次成功或单次变量组合不足以沉淀模板；需要多次稳定事件，且核心变量组合跨事件高度相似。候选实践路径的验证、晋升边界和与 ActivationLink 的区别见 `study.md` 第 7 节。

## 7. 总结

临时认知模型是知识大脑的工作记忆。

它让系统不是简单调用知识，而是围绕当前问题组织知识。

Working Model 是复杂问题的一次性思考结构，不需要默认完整保存。Working Model 内部推理过程不进入 Trace。

只有当 Working Model 的结果暴露了长期学习价值时，才产生 Learning Event，例如：

```text
某个知识缺口被确认；
某个 ActivationLink 被证明无效；
某组知识出现冲突；
用户纠正了 Working Model 的判断；
某类问题反复需要相同的知识组织路径。
```

Working Model 本身不是长期记忆，其完整内容不应直接沉淀。Study 只从 Learning Event 中抽取长期记忆调整信号，不从 Working Model 的完整推理过程学习。

一句话总结：

```text
临时认知模型把长期知识和外部证据组织成本次问题可推理、可检查、可追踪的工作结构。
```
